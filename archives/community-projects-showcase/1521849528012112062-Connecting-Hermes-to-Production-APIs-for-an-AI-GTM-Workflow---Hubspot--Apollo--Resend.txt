    Hey everyone!
    
    I've been exploring Hermes over the last few days, and I wanted to build something practical while learning the ecosystem.
    I ended up building a simple AI GTM agent. The workflow is pretty straightforward:
    
    - Fetch new contacts from HubSpot
    - Enrich them with Apollo.io
    - Generate a personalized outreach email
    - Send it using Resend
    
    For the execution layer, I used Swytchcode, so Hermes only has access to the APIs I expose. It also handles retries, policy enforcement, authentication, and audit logs, which makes experimenting with production APIs much easier.
    
    I recorded a short demo showing the entire workflow from start to finish.
    
    Demo: https://youtu.be/zKd98_0vR-E?si=Nk0BuJ66j5dvqC1U
    website link - https://dub.sh/s9dyIjc
    
    I'd really appreciate any feedback from people building with Hermes. If there's another workflow you'd like to see, let me know!

    Could you tell me why would you need Swytchcode when we already have a lot of other tools?

[2026-07-02T06:51:58.801000+00:00] charu1603 (id=1522132713686044693)
    We weren't trying to build another agent framework. We kept running into the same reliability problems once agents started calling real APIs, auth, retries, schema changes, rate limits, long-running workflows. Swytchcode exists to solve that execution layer, so developers don't have to rebuild it for every project.
