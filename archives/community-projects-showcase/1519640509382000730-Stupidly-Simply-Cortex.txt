    **Cortex: preflight + closeout for Hermes**
    
    Every agent workflow has one hidden failure point:
    
    **the moment before the agent starts planning.**
    
    That moment decides whether Hermes uses existing evidence or starts guessing again.
    
    When Hermes gets a dev/config task, the useful context is usually already somewhere:
    
    • local docs
    • solved fixes
    • exact commands
    • failed attempts
    • changed files
    • stale dependency notes
    • repo-specific rules
    • previous audit logs
    
    Most memory systems focus on **what the agent might remember**.
    
    Cortex focuses on:
    
    **what the agent must check before it is allowed to guess.**
    
    Before the task, Cortex runs a **preflight**:
    
    • search the doc library
    • inspect the right shard
    • check solved audit logs
    • detect stale docs
    • preserve changed files
    • rebuild indexes when needed
    • surface commands/evidence to reuse
    
    After the task, Cortex writes a **closeout**:
    
    • what was attempted
    • what worked
    • what changed
    • what command mattered
    • what model/harness info was exposed
    • what the next agent should reuse
    
    **Comparison:**
    
    • **Obsidian** gives humans notes. Cortex gives agents paths to check before acting.
    • **Honcho** gives user/peer memory. Cortex gives docs, commands, audits, and repo evidence.
    • **Mnemosyne / Holographic** help retrieve remembered context. Cortex adds canonical files, freshness, shard maps, and closeout logs.
    • **Vector/RAG systems** find similar text. Cortex connects text to source paths, stale status, changed files, and previous working commands.
    
    **Simple pitch:**
    Cortex helps Hermes solve tasks with local evidence first, then leaves a trail so the next run starts smarter.
