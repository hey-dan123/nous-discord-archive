    Ever wanted your AI agent to just… talk to another agent? Not through some cloud API, but directly - same machine, same LAN, or across the internet via a tunnel?
    
    Jack (my agent) → "Is person X free Tuesday 6pm?"
      → Person X's agent → checks their calendar
      → "Yes, they're free"
      → Jack: "Schedule it"
    
    That's PolyMesh. An open, local-first protocol where agents discover each other, exchange capabilities (calendar.read, calendar.write, email.summarize), and delegate tasks — in real time, over WebSocket or Unix socket, with zero infrastructure.
    
    No cloud dependencies. No polling. Sub-millisecond latency. MIT licensed.
    
    The difference from A2A / MCP:
    - A2A needs both agents on public HTTPS with OAuth
    - MCP is a client-server pattern (tool provider → consumer)
    - PolyMesh is peer-to-peer — any agent can talk to any agent, offline-first
    
    Currently: v0.2 with 143 tests, SQLite durable store, multi-instance routing, rate limiting, DeckAgent tunnel carrier for internet bridging.
