    Hey team! Just shipped something I've been cooking for weeks and I think a lot of you will love it.
    
      hermes-multichannel-prompt-optimizer → https://github.com/Sahil-SS9/hermes-multichannel-prompt-optimizer
     
     Sits invisibly in front of your agent and rewrites every prompt you send into something tailored to whatever model's listening.
     You keep typing how your brain wants ("hey could you maybe explain..."), the model receives something it
     actually knows what to do with. 
    
    Works with practically any model and across every Hermes surface (CLI, TUI, Discord, Telegram, Slack).
      
       Bonus: it logs every rewrite locally so /prompt-insights shows you exactly where your prompts are leaking tokens or quality. Honestly the analytics alone changed how I write.
      
       Tiny upstream PR for full CLI/TUI support → https://github.com/NousResearch/hermes-agent/pull/29526. Gateway
       surfaces work today on vanilla Hermes.

    What it covers
    
        | Language family                        | Detected by                                |
        |----------------------------------------|--------------------------------------------|
        | French, German, Spanish, Italian, etc. | langid                                     |
        | Arabic, Urdu, Hebrew (RTL scripts)     | langid + Unicode heuristic                 |
        | Chinese, Japanese, Korean              | langid + Unicode heuristic                 |
        | Russian, Ukrainian (Cyrillic)          | langid + Unicode heuristic                 |
        | Mixed-language prompts                 | langid returns dominant language           |
        | Very short prompts (< 10 chars)        | langid still works; heuristic returns None |

[2026-05-22T10:14:00.004000+00:00] sahil_ss9 (id=1507325651206803506)
    Hey team...   just pushed multi-language support to the prompt-optimizer plugin. If you type prompts in French, Arabic, Chinese, Spanish, etc., the optimizer now auto-detects the language and keeps the rewrite in the same language instead of translating to English. It uses langid under the hood with a fallback for non-Latin scripts.
    
        You'll need pip install langid if you want full coverage for all 97 languages. Without it, CJK/Arabic/Cyrillic etc. are still picked up via Unicode heuristics.
