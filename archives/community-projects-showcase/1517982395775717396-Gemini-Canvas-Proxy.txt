    # Running on a VPS with Tailscale
    
    Spin up a $5 VPS (any provider — Hetzner, DigitalOcean, Oracle free tier), install Chrome and Tailscale, then set up the extension and Canvas as normal. The native host binds to localhost:8765 by default, but since Tailscale creates a private mesh network between all your devices, any machine on your tailnet can reach the proxy at http://<vps-tailscale-ip>:8765/v1 — no port forwarding, no public exposure, no firewall rules. This is the "set it and forget it" deployment: the Chrome instance and Canvas tab live on a headless VPS running 24/7 (use Xvfb for the headless Chrome session), and your phone, laptop, or any Hermes Agent instance anywhere on your tailnet gets free unlimited Gemini inference without ever needing to keep a browser tab open locally. The proxy becomes a private API endpoint that follows you across every device.

[2026-06-21T18:34:05.459000+00:00] pranrichh (id=1518323139174596828)
    **Found and fixed.** Commit https://github.com/pranrichh/gemini-canvas-proxy/commit/5b22b47.
    
    ### The Bug
    The `elif bind_address == '0.0.0.0':` block I wrote earlier had `server_thread.join()` + `return` in it. When the proxy started in normal mode (not standalone) with `0.0.0.0`, it would:
    
    1. Start the HTTP server ✅
    2. Print a warning ✅
    3. **Block forever on `server_thread.join()` and never reach the native messaging code** ❌
    
    So `host_ready` was never sent to the extension, and the message-reading loop never ran. Canvas received the request and showed 200 OK, but the response had nowhere to go — curl just hung.
    
    ### The Fix
    Removed the blocking `elif` branch. Now it just logs the warning and continues to `send_message(host_ready)` and the main loop as normal. The `standalone` check gets its own early `return` so it doesn't interfere.

[2026-06-22T03:05:40.629000+00:00] pranrichh (id=1518451884049043528)
    Hermes-router is a load balancer that rotates many rate-limited keys to maintain uptime, whereas this proxy uses a single Gemini Canvas session to access Google's internal credentials which have no rate limits or daily caps. This approach provides a truly unlimited and free API for models like Gemini 3 Flash and Nano Banana 2 without the need to manage hundreds of keys or worry about 503 errors during heavy usage.
