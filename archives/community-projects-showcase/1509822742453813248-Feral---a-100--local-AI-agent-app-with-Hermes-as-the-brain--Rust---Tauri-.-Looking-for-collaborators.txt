    Hey 👋
    
    I'm building Feral — an open-source desktop app (Rust + Tauri + Svelte) that's evolving from a local LLM runner into a native Hermes Agent frontend.
    
    🔗 https://github.com/bloom500/feral
    
    What Feral aims to be:
    • LLM runner (like Jan) + Hermes Agent GUI
    • Persistent memory + cron scheduling + skill management
    • OpenClaw integration for MCP/tools
    • 100% local. Zero cloud. Zero account.
    
    Why not OpenHuman?
    → Requires account + cloud by default
    → No real scheduling (Hermes + cron destroys this)
    → No self-improving agent loop
    → OAuth to email/calendar/payments = security risk
    
    Current state: v0.1.0 — LLM runner works, agent layer is next.
    
    I've been running Hermes Agent with cron jobs in production for months (lead hunting, ads optimization). Now I want to give Hermes a proper native desktop home.
    
    Looking for: feedback, collaborators (Rust/Tauri/Svelte), and guidance on integrating Hermes Agent into Tauri.
    
    PRs, ideas, roasts — all welcome. <@387972437901312000> or any other staff member, I would love to hear your opinions, I will also need help implementing Hermes Agent natively into my app.

    🦊 Feral v0.2.0 — the agent finally opens its eyes
    
    Your private, local-first AI agent. Now with vision, real memory, MCP extensions, and a proper mascot. Windows · macOS (Apple Silicon + Intel) · Linux.
    
    📦 Download: https://github.com/bloom500/feral/releases/tag/v0.2.0
    
    ⚠️ Upgrading from 0.1.x — read this first
    
    The old update-signing key leaked into public git history, so it's been rotated.
    
    Coming from 0.1.7 or older → install the transitional 0.1.8 build first (it carries the new key), or grab the 0.2.0 installer manually one time.
    Coming from 0.1.8 → smooth automatic update.
    Onboarding, chats, models, BYOK keys — all carry over unchanged.
    Full plan: docs/UPDATER_KEY_MIGRATION.md
    ✨ What's new
    
    👁️ Vision that actually works. Pasted screenshots and uploaded images reach the model as real pixel data — not a [Image attached: …] filename stub. Works on OpenAI, Anthropic, and Ollama paths. Local GGUF stays text-only (the model can't see regardless).
    
    🧠 Memory that carries over. New chats no longer start cold. The chat tab now injects a memory context block into the system prompt on every send, and runs a background extraction pass that writes subject–predicate–object triples back into ~/.feral/memory-graph.json. "Remember X" / "forget Y" finally take effect immediately. Plus a redesigned Memory Graph page — dark canvas, glowing neon nodes, click-to-inspect details.
    
    🔌 MCP Extensions — native, secure. Feral now consumes Model Context Protocol servers through the official rmcp Rust SDK, fully backend-side. Curated catalog: File Access, Long-term Memory, GitHub, Web Search, Browser Automation, Deep Reasoning. One-click install, on/off toggles, "What can it do?" tool listing. No API keys, transports, or raw values ever reach the frontend.

[2026-06-12T20:44:29.637000+00:00] bloom9952 (id=1515094465365147738)
    🛠️ Stability (the unsexy part that mattered)
    
    - Stop button actually stops. Signal now flows end-to-end through a new Tauri command, sidecar stdin, and AgentLoop.stop() — aborts in-flight inference and running tools.
    - Agent tasks survive tab switches. - A live per-session mirror accumulates streamed content, tool bubbles, and phase even while another chat is on screen.
    - macOS/Linux: cloud model selection works in agent mode. 
    - Sidecar binary was being placed somewhere the host couldn't find it. Fixed.
    - GGUF chat template now reads from the model's own metadata via llama.cpp's template engine — renamed GGUFs no longer get a wrong template and corrupted output.
    - Real tokenizer endpoint — context accounting is no longer based on GPT-2 BPE guesses.
    - Stalled streams now report an explained error instead of silently ending in a fake "stopped by user" state.
    - Mid-reasoning dead ends — if a completion exhausts tokens while still thinking, the agent loop feeds the partial reasoning back and asks the model to finish (up to 4 continuations). No more "(model used all tokens on reasoning and produced no answer)".
    - Root React ErrorBoundary — render exceptions now show a recovery screen, not a blank window.
    - Humanized inference errors — 401/429/overflow mapped to plain English with a fix-it action. Raw error still under "Technical details".
