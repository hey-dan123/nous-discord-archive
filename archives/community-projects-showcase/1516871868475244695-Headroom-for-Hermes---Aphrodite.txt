    Hey <@584113584380969020> ran the plugin through codex when I hit a bug on the release version. Came back with the update fixing things, but also suggested a few changes. Our local fixes:
    Safe proxy process selection
    Replaced:
    lsof -ti :9797
    with listener-only lookup:
    lsof -tiTCP :9797 -sTCP:LISTEN
    Then verify /proc/<pid>/cmdline contains aphrodite before signaling it.
    Reason: the original command returns both the Aphrodite listener and Hermes client connections, causing Hermes to receive SIGTERM, then SIGKILL.
    Version normalization
    Normalize both expected and reported versions by removing the optional v prefix.
    Reason: proxy health reports 0.8.40, while BIN_VERSION is v0.8.40, causing a false mismatch and proxy restart on every session.
    Required CCR database argument
    Added:
    --ccr-db-path ~/.hermes/aphrodite/ccr-{cache|token}.db
    Reason: v0.8.40 refuses to launch without this argument, but the v1.62.55 plugin launcher omitted it.
    Fresh health checks
    Clear _alive_cache[port] before each readiness retry.
    Reason: otherwise _wait_alive() may return a stale cached result after restarting or failing to start a proxy.
