    I’ve started working on HermesBench, a practical eval suite for Hermes and personal AI agent configurations.
    
    Repo: https://github.com/verkyyi/hermesbench
    
    This is early/WIP. The goal is not another leaderboard benchmark. I’m trying to evaluate whether an agent setup actually works as a day-to-day operating system.
    
    The kinds of things I want to test:
    
    - front-desk behavior: answer simple questions directly, route real work correctly
    - Kanban/background task result return
    - cron/report quality without spam
    - AgentFeeds/current-context freshness checks
    - memory usefulness vs stale-memory mistakes
    - provider fallback behavior
    - delivery failure detection
    - privacy boundaries across DM/group/durable output
    - cost/latency/retry behavior
    
    My thesis: personal agents often fail in the surrounding system — routing, memory, freshness, notifications, retries, tool policy — not just in model reasoning.
    
    I’m collecting real-world workflows people think HermesBench should include.
    
    What’s one small repeatable scenario you’d expect a serious Hermes/personal agent setup to pass after config/model/profile changes?
