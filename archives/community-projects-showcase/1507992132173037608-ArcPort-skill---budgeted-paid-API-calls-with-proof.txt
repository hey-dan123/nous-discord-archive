    I built a small Hermes skill for testing paid tool/API usage with explicit budgets and proof.
    
    The skill connects Hermes to ArcPort, my own payment runtime experiment. The goal isn't to make Hermes "crypto-first". It's narrower: give an agent a controlled way to use paid APIs without unlimited spend, and without asking a human to approve every tiny request.
    
    **The flow:**
    1. Hermes opens a bounded session
    2. Hermes calls paid tools/APIs inside that budget
    3. each call updates cumulative spend
    4. Hermes closes the session when the task is done
    5. unused budget is returned
    6. the run can be inspected later by channel ID
    
    For the first demo, Hermes evaluates a few social trading signals and then calls Gemini for a short decision memo. But the trading example isn't the point. The point is the skill interface:
    - open a budgeted session
    - call paid APIs through that session
    - enforce allowed tools
    - track spend
    - close cleanly
    - return proof
