    Give one model many hands without splitting the brain
    
    Most multi-agent systems add more agents, but lose the intelligence that should stay in control.
    
    The strongest model should keep the actual goal, architecture, reasoning, tradeoffs, and final decisions. Workers should handle bounded research, implementation, verification, and command execution.
    
    That is what Sol Orchestrator does for OpenCode.
    
    Keep one brain. Give it many hands.
    
    Sol owns the outcome across multiple workflows. It can delegate jobs in parallel, see what workers are doing, steer them while they work, inspect only the result or diff it needs, reject weak output, retry or interrupt work, and revise the plan when new evidence changes the problem.
    
    Worker transcripts stay outside the main context unless Sol asks for them, so orchestration does not become context bloat.
    
    Goal: ship the complete feature
    │
    ├── Workflow 1: understand the real boundary
    │   ├── Graph v1
    │   │   ├── Step: frame the problem
    │   │   │   └── Job: Sol defines the questions
    │   │   └── Step: gather evidence
    │   │       ├── Job: worker inspects the runtime
    │   │       └── Job: worker maps affected callers
    │   └── Graph v2              replaces unfinished v1 after new evidence
    │       └── ...
    │
    ├── Workflow 2: implement the chosen design
    │   └── ...
    │
    └── Workflow 3: verify, integrate, and close the goal
        └── ...
    
    The harness includes durable goals, versioned workflow DAGs, supervised child sessions, review gates, steering, waiting, interruption, retries, targeted diff and artifact inspection, guarded undo/redo, compaction-aware state, and a TUI showing the graph, blockers, workers, review state, and valid next actions.
    
    The point is not a committee of agents. It is one capable model acting as architect and decision-maker, with several coordinated hands.
