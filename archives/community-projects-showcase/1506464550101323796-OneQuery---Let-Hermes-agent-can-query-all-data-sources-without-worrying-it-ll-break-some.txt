    I have a Postgres warehouse, a BigQuery project, a Mixpanel workspace, and a PostHog instance. Before this, asking my agent to "check yesterday's signups" meant giving it raw database credentials and hoping it wouldn't run something destructive.
    
    So I built **OneQuery** — one CLI to connect all your data sources with read-only enforcement, query cost limits, and a full audit trail. Then I wrapped it as a Hermes skill.
    
    **Setup takes about 30 seconds:**
    ```
    hermes skills install skills-sh/wordbricks/skills/onequery-cli --yes --force
    hermes chat --skills onequery-cli
    ```
    
    Now I can just say things like:
    - *"List my data sources"*
    - *"Use the warehouse source and get yesterday's DAU"*
    - *"Check the last 10 errors from Sentry"*
    
    The skill handles auth context, source discovery, SQL validation, and safe execution. For gnarly SQL, it writes to a temp file and uses `--file` instead of trying to shell-escape everything — a lesson I learned the hard way.
    
    Currently supports 15+ connectors: Postgres, Supabase, MySQL, MongoDB, BigQuery, Athena, Google Analytics, Amplitude, Mixpanel, PostHog, Sentry, GitHub, Linear, and more.
    
    It's open source (Apache 2.0, written in Rust) and self-hostable.
    
    🔗 GitHub: https://github.com/wordbricks/onequery
    🔗 Site: https://onequery.dev
    
    Feedback, bug reports, and new connector PRs all welcome.
