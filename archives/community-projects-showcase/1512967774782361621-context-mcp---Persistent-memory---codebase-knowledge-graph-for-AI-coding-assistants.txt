    The problem: Every AI conversation starts from zero. The assistant re-reads files it already read, re-discovers architecture it already understood, re-derives decisions already made. And if you switch tools — fix something in Claude Code, open Cursor — it knows nothing.
    What I built: A single MCP server that gives any AI assistant persistent memory and a structural knowledge graph of your codebase.
    What it does:
    
    Persistent memory — decisions, bugs, notes, and architecture saved across sessions and loaded automatically at conversation start
    Shared context store — lives at ~/.context-mcp/, not tied to any tool. Save from Claude Code, pick it up in Cursor, Gemini CLI, Codex, or Windsurf
    CodeGraph — AST-parsed knowledge graph (tree-sitter, 16 languages). Answer "what calls this function?" in less tokens instead of reading 10 files. 
    HTTP mode + OAuth 2.0 — connect Claude.ai and ChatGPT over HTTP with ctx online
    One install command — npm install -g context-mcp-server + ctx install --all
    
    Supports: Claude Code · Cursor · VS Code Copilot · Gemini CLI · Codex CLI · Windsurf · Claude.ai · ChatGPT
    Stack: Node.js (MCP server, CLI, HTTP/OAuth), Python + tree-sitter (AST extraction, NetworkX graph)
    🔗 https://github.com/vibhasdutta/context-mcp
    Would love feedback — especially on the CodeGraph query interface and the multi-AI context sharing. Still early, happy to answer questions.
