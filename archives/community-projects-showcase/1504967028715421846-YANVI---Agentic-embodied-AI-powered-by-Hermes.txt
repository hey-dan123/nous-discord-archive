    AI doesn't belong in a tab.
    
    YANVI is Agentic Embodied AI powered by Hermes: an always-on physical presence for knowledge workers, with an expression engine, screen awareness, memory, voice, and desktop autonomy.
    
    The idea is simple: agents should not only live inside chat windows. They should be able to understand your workspace, show what state they are in, remember context, and act across your Mac when you ask.
    
    Hermes powers the agent layer behind YANVI: tools, memory, skills, workflows, and desktop actions. YANVI gives that intelligence a body, a face, and visible state.
    
    This thread will be the main YANVI build log inside the Hermes community.
    
    I’ll share:
    - live demos
    - architecture notes
    - local-first experiments
    - expression engine progress
    - screen awareness and Mac autonomy work
    - failures and fixes
    - Kickstarter prep for October 2026
    
    Current thesis:
    
    AI thinks. YANVI feels.

    YANVI now has Live Mode via gpt-realtime-2, with the agent layer still on Hermes for tools, memory, and Mac actions.
    
    So the split is:
    - realtime voice for fluid conversation
    - Hermes for everything that touches the world
    
    The result feels near-zero latency. Demo below.
    
    Shipping this as “YANVI Live Mode” in the Kickstarter unit. Default mode stays local-first on Hermes. Live Mode is opt-in with your own OpenAI API key, and you choose live vs default every time.
