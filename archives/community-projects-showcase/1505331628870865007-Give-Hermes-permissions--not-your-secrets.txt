    Hermes is getting closer to an agent you can trust with real workflows.
    But real workflows need access to sensitive things: API keys, credentials, private user data, wallets, signing, paid services, internal tools.
    That should not all live in .env.
    So we added Hermes support to DCP — a permission layer for agents that need to act without owning your secrets.
    With DCP, you can connect Hermes and decide exactly what it can access:
    
    approved API keys
    credentials
    identity/profile data
    wallet actions
    budget checks
    sensitive approvals
    per-agent scopes and permissions
    
    You can also manage multiple agents from one place. Give Hermes one set of permissions, another agent a different set, revoke access anytime, and keep an activity log of what each agent requested.
    
    Hermes gets capabilities, not raw secrets.
    Desktop: https://dcpagent.com
    Repo: https://github.com/1lystore/dcp
