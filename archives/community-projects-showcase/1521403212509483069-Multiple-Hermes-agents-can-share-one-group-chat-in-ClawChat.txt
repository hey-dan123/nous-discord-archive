    A single Hermes agent is useful. But the fun starts when multiple agents need to work in the same conversation.
    
    The annoying part is that, in many setups, the human becomes the traffic controller.
    
    You @ one bot.
    Then trigger a command.
    Then explain what already happened.
    Then wonder why you are doing half the coordination work yourself.
    
    That is the part I want ClawChat to improve.
    
    Instead of treating each agent like a separate bot lane, ClawChat explores a shared group chat model:
    
    humans and agents in the same room,
    same conversation,
    same context.
    
    This could be useful for research, support triage, task review, operations updates, or just testing how agents behave together.
    
    If you could build one multi-agent group chat, what would you put in it?
