    Hi everyone. I built a small project around Hermes Agent called Hermes Jail.
    
    It is not Docker support for Hermes, since Hermes already has Docker/backends. My goal was a bit different: I wanted a controlled workbench where Hermes can install tools, try things, work inside a persistent "/workspace", serve local web output through nginx, and make a mess without making that mess directly on my host.
    
    I come from a DevOps/infrastructure background, so my focus was not only “can the agent run?”, but also where it runs, what it can touch, what persists, and how to keep real infrastructure separated from the agent’s experimental work.
    
    The basic idea is simple: the container is where the agent experiments. Real infrastructure actions should go through controlled interfaces or brokers instead of giving the model raw host access, SSH keys, cloud credentials, or the Docker socket.
    
    Hermes Jail includes localhost-only web exposure by default, no privileged mode, no host network, no Docker socket mount, a persistent web root under "/workspace/web", optional firewall helpers, a seeded environment skill, and a small threat model.
    
    It is not a hardened sandbox, and I’m not claiming it makes arbitrary code safe. It is just the baseline I wanted for running a persistent agent in a useful but more controlled local environment.
    
    Repo: https://github.com/ivanneira/hermes-jail
    
    I’d appreciate feedback from Hermes users, especially on whether the framing is fair, whether this overlaps too much with existing Hermes Docker/backend patterns, and whether any default looks like an obvious footgun.
