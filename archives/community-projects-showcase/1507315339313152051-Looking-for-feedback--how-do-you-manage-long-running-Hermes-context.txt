    Hey everyone..!!! 
     I’ve been building something for my own Hermes workflow, and I’d love feedback from people who run longer agent tasks.
    
    The problem I kept running into is context continuity.
    
    When I work with an agent for a long time, pause, come back later, or split work across subagents, the chat is technically still there but the actual work context often gets blurry.
    
    Things like:
    1.  decisions we already made
    2.  criteria we agreed on
    3.  prompts or skills that worked well
    4.  handoffs to subagents
    5.  why we chose one direction over another
    
    The tricky part is that even when the context is lost, the agent can still continue confidently — just not always in the same direction.
    
    So I started building Alexandria-Hermes.
    prompts, skills, decisions, handoffs, memory compacts, and captured context that Hermes or other agents can recall later.
    
    The goal is not to build another autonomous agent runtime.
    I’m trying to make AI collaboration less fragile when work spans multiple sessions, pauses, and handoffs.
    
    I also added a librarian idea:
    if Hermes needs a skill or context that isn’t available yet, the librarian can help search, draft, or prepare candidates while Hermes keeps working.
    
    I built it first because I personally needed it  but I’m curious if this resonates with others here.
    
    I’d love to know:
    1.   Do you run into this context-continuity problem too?
    2.  How are you currently managing long-running Hermes workflows?
    3.  How do you store prompts, skills, decisions, and handoffs today?
    4.  Would something like this be useful in your workflow?
    5.  What feels missing or unclear?
