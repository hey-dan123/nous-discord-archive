    Updates:
    
     **hermes-mcp updates**
      _(the MCP bridge that lets any LLM client delegate tasks to your local Hermes Agent)_
    
      **Repo renamed.** `claude-hermes-mcp` → `hermes-mcp` to reflect that it's not Claude-only anymore. New URL: https://github.com/mlennie/hermes-mcp — old links auto-redirect on GitHub     but please update bookmarks and remotes.
    
      **Async jobs (v0.3).** Long-running tasks no longer hit the MCP client's ~2-min per-tool-call timeout. Pass `async_mode=true` to `hermes_ask` and you get back a `job_id` immediately;
      poll with `hermes_check(job_id)`. Hermes keeps running in the background whether you poll or not. Great for browser-driven work, multi-step agentic chains, anything >30s.
    
      **Job cancel + reset (v0.3 / v0.4).** `hermes_cancel(job_id)` releases the bookkeeping on a stuck job; `hermes_reset()` wipes the whole queue. Important caveat: cancel/reset don't stop   already-running worker threads — side effects in flight (emails sent, files created) still happen. Use them to release results, not undo work.
    
      **LLM agnostic (v0.4 + v0.5 in review).** Server now supports two auth paths in parallel: PKCE-only OAuth and a static bearer token (`MCP_BEARER_TOKEN`, opt-in). Tested end-to-end
      against Claude (web/desktop/mobile), Codex desktop, and Cursor. v0.5 lands when PR #5 merges.
    
      Issues / config questions welcome → github.com/mlennie/hermes-mcp/issues
