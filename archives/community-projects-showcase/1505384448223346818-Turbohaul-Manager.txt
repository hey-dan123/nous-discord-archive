    Ollama-shape inference manager using Tom's TurboQuant fork of llama.cpp.
    
    FIFO queue + grace + IDLE_HOT hot-hold + model swap on Nvidia RTX GPU's including Blackwell.
    
    
    
    What it does
    Accepts OpenAI / Ollama-shape /v1/chat/completions requests
    Single-slot serial sidecar (one llama-server child holds the model)
    ACTIVE_MATCH cascade for same-thread follow-ups within a grace window (warm-process reuse)
    IDLE_HOT 5-minute warm-hold after grace expires: same-model follow-ups inherit the warm process; different-model swap tears down + spawns new
    Safety guardrails: refuses spawn when VRAM / RAM / CPU / IO-wait would put the host at risk

    v 0.4.0 now available! 
    https://github.com/MrTrenchTrucker/turbohaul-manager/releases/tag/v0.4.0
    Highlights
    Live inference monitor — a built-in, real-time view of the active generation: tokens/sec, prompt/decode progress, and a live output stream (SSE) — independent of any external client or logger.
    Per-model concurrent dispatch — a main chat model can run in series while a sub-agent model serves up to N requests concurrently (parallel: N) on a single warm llama-server sidecar. The single-model-resident invariant is preserved.
    KV-cache offload to host RAM — with no_kv_offload, model weights stay on the GPU while the KV cache lives in system RAM. Run a large context (e.g. 250K) on a 24 GB card, and — with kv_unified: true — serve requests in parallel while offloaded. See docs/KV_CACHE_OFFLOADING.md.
    Live-panel stability + tool-call display fixes — no more flicker between "generating" and "idle" during bursty generation; tool calls render correctly in the live output.
    Faster failure recovery — a crashed model load fails its slot in ~2 s instead of waiting out the full load-health timeout, so a bad load no longer wedges the queue.
