    I built a small Hermes-centered research workflow skill for structured game project discovery and technical review.
    
    The idea comes from a real community need: gaming communities often find promising new projects too late, after the early testing and learning window has already passed.
    
    This workflow helps Hermes Agent organize early project research by reviewing public development signals, repository activity, documentation quality, project notes, and clear next-step classification.
    
    Current version includes:
    - Hermes workflow skill foundation
    - daily report template
    - structured research process
    - public repository signal review
    - documentation quality checks
    - project notes
    - WATCH / TEST / CONTACT / SKIP classification
    - Discord-ready report format
    
    The workflow is designed so Hermes can later:
    - run this research process on a schedule
    - remember previous reports
    - compare new findings with older reports
    - generate clean community-ready summaries
    - support a future scanner and scoring system
    
    I kept this first PR intentionally small and easy to review: skill foundation + report template only.
    
    This is only a research and workflow organization tool. It is not related to any asset, promotion, or external claim.
