    **ContribScout v0.2.1: Stability / Bugfix Update**
    
    Small update for ContribScout today.
    
    After testing the live version, I noticed a few things that needed cleanup:
    • suggested actions were repeating too much
    • localization-based actions were showing up more often than they should
    • refreshing the dashboard could sometimes fall back to sample data
    • live GitHub scan behavior needed to be more stable
    
    Updated:
    • more varied suggested contribution actions
    • better signal-based action selection
    • localization suggestions are now used only when they actually make sense
    • improved live GitHub scan stability
    • added safer caching for successful GitHub results
    • reduced unnecessary fallback to sample data
    • clearer source / notice handling

    **ContribScout v0.3 Update**
    
    Added a few polish and usability updates today.
    
    What changed:
    
    • added Proof Vault export tools
    • Markdown export for saved contribution proof
    • JSON export for backup / restore workflows
    • copy-to-clipboard report action
    • improved header branding with a cleaner Hermes Skill Layer badge
    • cleaned up roadmap and dashboard copy
    • kept the live GitHub scanner and Hermes skill package intact
    
    The main goal of this update was to make Proof Vault more useful.
    
    Instead of only saving planned/submitted contribution actions locally, you can now turn them into a clean report that can be reused for updates, applications, or personal contribution logs.
    
    ContribScout is still a standalone dashboard, with a Hermes-compatible skill layer included in the repo for daily contribution report flows.
    
    I’d really appreciate it if you could try the site and share any feedback.
