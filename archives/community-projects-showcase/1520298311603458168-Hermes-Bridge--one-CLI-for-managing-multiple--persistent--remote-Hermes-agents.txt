    I live in my terminal, and I love the TUI.
    
    I also have a bunch of agents running on remote machines with intentionally separate scopes & auth (eg, one personal agent, another to manage a side-business of mine, another for my wife, etc...).
    
    I _hate_ sessions getting interrupted by a lost ssh connection because my macbook decided to turn on screen lock.
    
    So, I built an as-simple-as-possible CLI shim to fix all that:
    https://github.com/carpdiem/hermes-bridge
    
    It's a small CLI that makes remote `tmux` the home base for each Hermes agent; your laptop is just the window into that session.
    
    When you type config-registered agent names like `personal`, `business`, or `family`, Hermes Bridge SSHes to the right machine/account and starts or reattaches to the right remote `tmux` session.
    
    So each agent can keep its own machine, auth, inboxes, and config, but a sleeping laptop or dropped SSH connection doesn't kill the work—you just reconnect and pick up where you left off. And all the while, you've only got a single, unified interface to deal with.

[2026-06-30T19:15:46.624000+00:00] carpdiem (id=1521595120611491901)
    i actually don't use profiles. i literally just have a separate mac os user account for each hermes agent, so each has their own filestystem permissions / user home directory / etc
    
    so i've got macos users like: admin, hermes-personal, hermes-llc, hermes-family, hermes-wife
    
    then a lot of the specialization comes from really extensive work with the SOUL.md & USER.md, where I've actually exported all of my previous AI tool usage from, eg, OpenAI/etc and had the relevant hermes agents go over and analyze all of it to extract relevant things for their own domains.
    
    each agent gets some extra custom treatment: individual telegram bots, per-agent specified google workspace access (eg, my 'hermes-llc' gets access to the google drive workspace for my side-hobby llc, but my 'hermes-personal' gets access to my personal google drive/etc)
    
    and since it's on a dedicated mac mini, i also use dropbox folders for easy file sharing & collaboration with my laptop. so while each agent gets its own user home directory to play in, it also gets a single folder shared from my dropbox, like ../Dropbox/hermes-personal/ with some subdirectory structure inside of it so i can drag-and-drop files to share from my laptop, or automatically get new files (that the agents generate) synced right back to my laptop
