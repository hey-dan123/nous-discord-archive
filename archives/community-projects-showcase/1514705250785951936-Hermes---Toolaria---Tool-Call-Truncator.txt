    Toolaria 🛟 — stop oversized tool results flooding your context
    
      Built a little Hermes plugin. MCP and web tool calls (web_extract, browser snapshots, big JSON) dodge the normal truncation and dump 50-200k chars straight into context. Toolaria catches them at the
      transform_tool_result hook, spills the full thing to a SHA256 blob store, and hands the model a compact excerpt plus a fetch handle.
    
      The model then pulls only what it needs:
      rescuer_fetch(id, mode="grep"|"range"|"stat"|"full")
    
      It's the spill-to-disk pattern Claude Code uses, packaged as one zero-config plugin. On by default, no engine swap, composes with whatever context setup you run. Does one thing.
    
      
    SHA256-addressed, dedupes identical results
    grep runs under a real timeout (no ReDoS foot-guns)
    swept blobs leave a breadcrumb telling the model to re-run the source tool, not a dead handle
    
      Clone, enable, hermes plugin reload. That's it.
    
      👉  https://github.com/Sahil-SS9/Toolaria
    
      Feedback welcome, especially if you run it next to a heavier context engine

    Toolaria v2 is live
    
    Quick recap: big tool results (web pages, API dumps, MCP output) used to flood the model's context and crowd everything else out. Toolaria catches them, stashes the full thing on disk, and hands the model a tiny handle instead.
    
    v2 turns that handle into a proper workspace:
    
    Outline — get a structural map of a result without reading it. JSON schemas with number stats, page headings, log error clusters. The model navigates by structure, not guesswork.
    
    Search — ask a rescued result a question in plain English, get the relevant bits back with line numbers. Works dep-free out of the box, smarter with embeddings installed.
    
    Pass-by-reference — the big one. Hand a whole result to another tool without the model ever reading it. Write tla:<id> and Toolaria swaps in the full content right before the tool runs.
    
    The numbers, on a real 2,000-row result:
    - 171,874 chars became a 1,733-char handle. 99x smaller in context.
    - Passed it to a second tool by emitting 79 characters. The tool still got all 171,874.
    
    Hardened too: session-scoped so one session can't pull another's data, exec/exfil sinks blocked by default, grep that can't be hung by a nasty regex.
    
    Zero config, on by default, composes with whatever context engine you run.
    
    Repo and a Proof section showing it in action: https://github.com/Sahil-SS9/Toolaria
