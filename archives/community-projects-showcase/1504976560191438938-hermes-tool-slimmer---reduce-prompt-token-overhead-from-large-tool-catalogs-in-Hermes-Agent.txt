    Built a small plugin for Hermes that trims the tool schema list injected into each turn, rather than passing the full catalog every time.
    
    The problem: A 57-tool installation serializes to ~73 KB / ~18K tokens per call. Most of those tools are irrelevant to any given turn.
    
    What it does:
    - Builds a BM25-ranked index of your tool schemas
    - Selects 8–12 relevant tools per turn based on the current prompt
    - Keeps configured safety tools (always_include) hot regardless of ranking
    - Supports Anthropic's native Tool Search path (mode: anthropic_tool_search) where available
    - Falls back to deterministic keyword or hybrid selection for OpenAI/OpenRouter/local providers
    - fail_open: true by default — any selector error preserves the original full schema list
    - dry_run mode for validation before you commit to live schema changes
    
    Includes a minimal upstreamable patch artifact for the Hermes core selector hook (docs/hermes-core-selector-hook.patch) for anyone who wants to contribute the surface upstream.
    
    Install:
    pip install "git+https://github.com/alias8818/hermes-tool-slimmer.git@v0.1.0"
    hermes plugins enable tool-slimmer
    hermes tool-slimmer status
    
    GitHub: https://github.com/alias8818/hermes-tool-slimmer
    
    Early release — feedback on the core hook contract and provider path behavior especially welcome.

    Hermes gave me this suggestion. If you could somehow build a function in to make this easy that would be fantastic. Native "Description Minification"
    Standard tool schemas are often written for humans, containing verbose sentences like "This tool is used by the agent to carefully read the contents of a file from the local filesystem provided a path..."
    
    The Move: Rewrite your tool descriptions in Telegraphic Style.
    
    Before: "A tool that allows you to execute shell commands in the terminal." (14 tokens)
    
    After: "Run shell commands." (3 tokens)
    
    Why it works: hermes-tool-slimmer still has to include the descriptions of the top_k tools it selects. If your 8 selected tools have long-winded descriptions, you're wasting tokens. Shaving 20 tokens off each description saves 160 tokens per turn—which adds up to thousands over a long session.
