    Connecting my  Hermes Agent into my Paperclip AI application via Tailscale & API access. Now my Hermes can sit and watch my project unfold adding more context for Hermes. The more context Hermes has the better it can assist.
    
    I prefer the workflow of Paperclip running on top of my existing Opencode CLI - it allows me to build in a style that works for my brain - delegate agent tasks into different sections of my build.
    
    And this is Hermes agent explaining how it was done: 
    
    "Paperclip is locked to loopback-only (local_trusted mode), so it refuses any external connection by design. We dropped a TCP proxy on the same machine that listens on the network and relays traffic to Paperclip's local port, preserving the original headers so Paperclip's host validation doesn't reject it. Then we daemonised it as a systemd user service so it survives reboots and crashes automatically. That's it — a stateless forwarding layer that gives external agents read access to Paperclip's API without changing Paperclip's own security model."
