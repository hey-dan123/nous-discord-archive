    Here's a quick guide to getting up and running with fmsg using https://fmsg.io with hermes - based on the open protocol: fmsg, for threaded conversations. Self-hosting is also possible if you have a domain name, static ip and familiar with docker, the OSS stack is: https://github.com/markmnl/fmsg-docker. So agents can directly message each-other or anyone with a fmsg address on the internet.
    
    We will use hermes-fmsg, and hermes platform plugin announced today here: https://discordapp.com/channels/1053877538025386074/1528445678630469715/1528445678630469715 
    
    1. Create an account on https://fmsg.io e.g.: `@alice@fmsg.io`
    
    2. Create a sub account after logging in Admin > Agents > New agent. You will need:
        * `FMSG_API_URL`
        * `FMSG_API_KEY`
    
    3. Run on your machine with hermes installed:
    
    👉  `hermes plugins install markmnl/hermes-fmsg --enable`
    
    4. Supply your fmsg address, the API url and API key to configure
    
    5. Enjoy threaded messaging where threads form durable contexts hermes will resume from at any time! 
    
    
    Any issues please shout! Been working on fmsg a while now and this is the first real use-case. 
    
    If you self-host fmsg on your own domain can have direct agent-to-agent messaging with no limits except what you server can handel. Since fmsg is extremely efficient (my background in protocol design is game dev net code), thats a lot!
