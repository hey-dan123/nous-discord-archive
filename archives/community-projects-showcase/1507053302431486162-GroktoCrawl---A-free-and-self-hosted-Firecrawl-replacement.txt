    FYI, I'm thinking Exa & Firecrawl are each kinda ok on their own. But what if they had a baby with the best features of both parents?

    # GroktoCrawl v0.4.0 — 2026-05-31
    *GroktoCrawl is a self-hosted, API-compatible Firecrawl alternative with Agent endpoint. MIT licensed. One docker compose up and you're running.*
    
    ## 14 files changed, 1,341 insertions, 274 deletions since v0.3.0.
    
    ## CLI
    
    - **monitor, parse, generate-llmstxt subcommands** — three new CLI entry points (#79, #81)
    - **50x search speedup** — removed redundant per-result scraping from /v2/search (#69)
    - **Search format fix** — CLI reads from data.web per Firecrawl v2 spec, no more silent empty results (#71, #66)
    
    ## API
    
    - **Agent system prompt upgrade** — comprehensive research prompt with source authority ladder, contradiction detection, and citation requirements
    - **Extract prompt upgrade** — extracts ALL instances, flags missing values
    - **Model selection passthrough** — model field in agent requests now respected, overrides env default
    - **Domain metadata** — each source tagged with (domain: ...) for credibility evaluation
    
    ## Documentation
    
    - **Skill document overhaul** — comprehensive workflow-oriented skill doc spanning 4 SkillOpt epochs: browser lifecycle, structured extraction, multi-source fallback chain, domain exploration strategy, change monitoring, error recovery across all commands
    - **Decision tables** — "When to use which" and tool selection guides

    Groktocrawl has an *adapters* framework in `main` branch now. So far we've got adapters for *YouTube* and *Bluesky*. 
    
    For specific site URL's, `groktocrawl scrape` will use a specialized adapter. So like, if you try to scrape a YouTube link (even one of their shortlinks), it will return back something that kind of looks like an Obsidian note. YAML frontmatter with metadata about the video, and Markdown transcript.
