    OverCR is a Hermes-first portable AI orchestration substrate built for long-lived, governed AI operations instead of disposable chat sessions. Filesystem state is canonical, workflows are auditable, model output is treated as untrusted until validated, and recovery paths are designed in from the start. The project focuses on operational continuity, governed subagents, workflow choreography, deterministic fallback behavior, and cold-start reconstruction from filesystem artifacts alone.
    
    The current release includes four bounded specialist subagents — KnowER, CryER, CodER, and PypER — operating through layered L1-L6 validation, approval-aware governance, packetized routing, audit trails, and Hermes-backed inference infrastructure. CodER and PypER remain advisory-only by design, while workflow execution stays explicitly constrained and operator-controlled. The entire system is runtime-agnostic but Hermes-first, deployment-portable, and built around inspectable filesystem-native state rather than opaque session memory.

    is the "Canonical state lives on disk"  different than the native hermes sessions log file which contains session history ? or are they the same thing ?

[2026-05-12T16:52:37.814000+00:00] guideboardlabs (id=1503802091116953751)
    They’re related, but not really the same thing. Hermes session logs are basically conversational memory — they remember what was said, what tools were called, the flow of the interaction, etc. OverCR canonical state is more like the operational truth of the system itself.
    
    So if Hermes is “the conversation,” OverCR is “the actual system state.”
    
    The filesystem-first idea came from not wanting the orchestration layer to depend entirely on chat continuity. Sessions can drift, truncate, fork, or lose structure over time. OverCR is designed so that even if the conversation history gets messy, the system can still reconstruct itself correctly from the operational artifacts on disk — tasks, packets, workflow traces, audit logs, manifests, freezes, all of that.
    
    So the Hermes logs are definitely useful as reconstruction context, but they’re not considered authoritative state by themselves.
