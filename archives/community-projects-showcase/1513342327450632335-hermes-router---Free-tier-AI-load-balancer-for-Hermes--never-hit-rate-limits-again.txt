    Hey everyone 👋
    
    Built a lightweight proxy that sits between Hermes and your AI providers — so when one free tier hits a rate limit, it automatically falls through to the next provider and rotates keys. No downtime, no manual switching.
    
    The problem it solves:
    Gemini keys hit per-minute limits. OpenRouter has a 50 req/day cap. Groq throttles. If you're running Hermes 24/7 or sharing it with others, you're constantly hitting walls.
    
    How it works:
    Your Hermes points to hermes-router instead of directly to a provider. It handles everything:
    
    → Gemini (rotate across multiple keys)
    → OpenRouter (rotate across multiple keys)
    → SambaNova
    → GitHub Models (free with any GitHub account)
    → Cerebras
    → Groq
    
    Rate limited? Tries the next key. All keys cooling? Cascades to the next provider. Completely transparent to Hermes.
    
    Also includes:
    Strips reasoning_content / think fields that Gemini outputs but Groq/Cerebras reject (prevents 400 errors during fallback)
    Response cache — identical requests skip the provider entirely
    /v1/status endpoint showing live key health + latency stats per provider
    Optional fast routing — short messages go to low-latency providers first
    
    Stack: Single-file Python (Flask) — ~300 lines, easy to read and fork.
    
    🔗 GitHub: https://github.com/Shaf2665/Hermes-router
    
    Happy to answer questions or take PRs if anyone wants to add more providers!

    # Update — Smart Routing just landed 🧠
    
    hermes-router can now pick the right model for each request automatically.
    
    ## How it works:
    On startup, the router probes every provider, validates models (and auto-fixes broken ones by querying the provider's /models API), then assigns each a capability rating:
    1 — outstanding   gpt-4o, claude-opus, gemini-pro
    2 — best          nemotron-120b, llama-70b, gemini-flash
    3 — good          gpt-4o-mini, mistral-small, llama-8b
    4 — fair          command-r7b, smaller models
    
    https://github.com/Shaf2665/Hermes-router
    
    
    Every request is then classified by complexity (token count + keyword signals) and routed to the **cheapest model that can handle it** — no wasted quota on a powerful model for a simple question.
    
    **Simple request** → cohere/groq (fast, cheap)
    **Complex request** → openrouter/sambanova (strongest free models)
    
    Zero added latency — classification is pure heuristic, no extra API call.
    Results persist to `router_state.json` so ratings survive restarts.

    # hermes-router — update
    
    **Circuit breaker + health-aware routing**
    
    If a provider keeps throwing errors or timing out, it gets skipped automatically for ~60s, then probed again
    Healthy providers get picked first — flaky ones sink in the order
    Wasted round-trips on broken providers are basically eliminated now
    
    Why no token compression?
    A few people have asked about adding RTK/Caveman-style compression. We looked at it — but since Hermes Agent already handles compression on its end, adding it to the router would be double work. On top of that, the router is free-tier only, so there's no cost saving. We'd just be adding risk of corrupting messages for zero benefit.

    # New: key rotation modes ⚙️
    
    You can now choose how hermes-router picks between multiple keys/accounts on a provider:
    round-robin (default) — spreads requests evenly across all your keys. Best for load + latency.
    
    sequential — drains one key fully until it rate-limits, then moves to the next. Keeps your later accounts fresh in reserve instead of burning them all at once — great for rationing accounts after a quota reset.
    
    `hr mode              # see current mode`
    `hr mode sequential   # switch`
    `hr restart           # apply`
    
    Default is unchanged, so nothing breaks if you don't touch it. The active mode shows up in hr status. Failover, cooldowns, and the circuit breaker all keep working — this only changes which key gets preferred next.

    ## 🐛 Bug Fix Update — thanks for the reports!
    
    Two issues found by the community, both fixed and pushed:
    
    1. **hr setup/hr auth/hr model/hr mode** failing on macOS 🍎
    Reported by <@207041642224549888> — macOS's default bash (3.2) doesn't support the lowercasing syntax our scripts used, causing bad substitution and Unknown provider errors. Fixed with a portable alternative that works on any bash version.
    
    2. OpenCode Go always erroring 🔧
    Reported by <@215303553768488970> — turns out opencode_go needs Go billing actually enabled on your OpenCode account (having the key alone isn't enough — it's just the router's signal that you've subscribed). We also found and fixed a real bug while investigating: a provider stuck with a bad/unsubscribed key was getting retried on every single request forever instead of backing off. Now it cools down properly like any other unhealthy provider.
    
    Pull the latest with `hr update` (or git pull + restart) to get both fixes. 🙏

    ## 🔑 New on Hermes-router: Access Keys page + provider-scoped keys
    
    
    **Been expanding the dashboard's key management:**
    
    • Access Keys page — mint new proxy keys straight from the dashboard for teammates/other apps, with optional rate/budget limits (req/min, req/day, tokens/day, cost/day). Full key is shown once in a copy box, then only the last 6 chars ever again. Edit or revoke keys in place, no restart-and-pray.
    
    • Provider scoping — you can now restrict what an access key is allowed to route through. Pick specific providers when creating/editing a key (checkboxes show live "X keys" counts per provider) and that key's requests will only ever hit those providers — never falls back to anything outside its scope.
    
    • Safer defaults — since a proxy key now unlocks real dashboard config-write power, the router no longer lets that sit on the shared placeholder value. If you never set your own, it generates a real random key on first boot and saves it to .env for you, logged once so you can grab it.
