    Hey folks - Kushal here. I've been working on Stratum, a local-first multi-agent orchestrator that runs entirely on your hardware. No cloud backend, no telemetry, no per-seat pricing.
    
    Architecture-wise, it's a manager-agent topology running in an Electron shell. The manager owns planning, delegation, verification, and recovery. Workers (frontend, runtime, test, browser, review) each get bounded tasks with visible state and dedicated tool surfaces. The manager checks diffs, logs, screenshots, and summaries before accepting work, with automatic retry on failure.
    
    The agent runtime supports 75+ LLM providers including Nous Hermes through the OpenCode integration and direct provider configs. Model routing is per-worker - you can assign different models to different workers. Context windows are tracked per run, and project memory (plan.md, codebase.md, notes) persists across sessions so long-running work doesn't restart cold.
    
    It's been battle-tested on my own research work - edge efficiency, scheduling, AI tutoring pipelines - running on CPU-only hardware with intermittent connectivity. The whole thing stays under 128K tracked context per run with durable artifacts written back into the project directory.
