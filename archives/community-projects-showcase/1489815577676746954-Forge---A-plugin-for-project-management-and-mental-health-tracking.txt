    Hey guys 👋
    
    I’ve been building Forge, an OpenClaw and Hermes agent plugin that gives agents a more structured workspace for ongoing work and psychologically relevant state.
    
    Forge is meant to complement the usual open-ended natural-language memory and reasoning. That kind of memory is great for nuance, rich context, and flexible expression. Forge adds a more explicit operational layer alongside it, so agents can persist, inspect, update, and act on structured units of work and mental-health-related state over time.
    
    With Forge 0.2.19, the plugin now supports:
    
    - goals, projects, tasks, and live task tracking
    - Kanban state
    - agent insights
    - a gamified progress layer with XP
    - habits
    - structured notes
    - event management
    - iCloud and Google Calendar sync
    
    It also includes a mental health module for working with structured records like values, beliefs, patterns, modes, and self-observation reports.
    
    That part is informed by 3rd wave CBT, especially ACT and Schema Therapy. The goal is practical rather than clinical. It is meant to help the agent keep track of what matters, what keeps recurring, what tends to drive avoidance or misalignment, and which actions are more coherent with values and longer-term direction.
    
    In practice, Forge lets agents manage goals, projects, tasks, habits, notes, and events as explicit objects and connect psychological patterns to concrete work, decisions, and next actions. You can work either through chat or through a dedicated Forge UI when visual inspection is more useful.
    
    The feedback I’d most value is:
    - what seems genuinely useful
    - any installation, setup, or onboarding friction
    
    I’d love to collaborate with anyone interested in the project. Thanks 🙂

[2026-06-07T21:00:05.208000+00:00] _albertjames (id=1513286450106339569)
    Forge 0.3.0 is released 😬🚀
    
    The main change since the previous 0.2.6 release line is : training, weight loss, nutrition, and improved install/runtime flow.
    
    Forge now has workout/training analysis. It can use HealthKit workout history to rebuild richer workout evidence, show training load, and reason about training zones.
    
    Has weight-loss and nutrition tracking. You can log food, add custom foods, see weight-loss insights, check whether the data is reliable, and calculate calorie targets from recent activity instead of only using a static estimate.
    
    Handles much larger HealthKit history imports. The existing iPhone sync line now supports compressed workout archives, resumable uploads, backend progress, and deeper historical workout replay.
    
    Has alpha WatchOS foundation. The Watch side can receive Forge snapshots and commands, send actions back, and work through the iPhone relay.
    
    Has improve onboarding and install with standalone installer/runtime package: forge-memory. It sets up Forge, the data folder, OpenClaw, Hermes, Codex MCP, health checks, repair tools, and iPhone pairing from one place.
    
    Phone setup is more practical now. Forge can pair through Tailscale, Iroh, or a fixed/private IP instead of relying on links that only work on the host machine.
    
    Agents and plugins are also cleaner: OpenClaw, Hermes, Codex, and Forge Memory now connect through the same Forge runtime more consistently.

    Forge 0.3.19 update 😗 
    
    Main changes since 0.3.0: Artifact Store, much better iPhone/Watch sync, Claude Code support, safer installs, and cleaner agent behavior.
    
    Forge now has an Artifact Store. You can save images, PDFs, docs, spreadsheets, and other files with title, description, provenance, safety state, danger score, links, versions, and audit history.
    
    Uploads now use a guided modal flow. You can add several files at once, fill details only where needed, use LLM enrichment, and link artifacts to normal Forge entities. Agents can help with metadata and scans, but cannot autonomously open, run, or download file bytes.
    
    Artifact Store also got delete/archive and pagination, so large collections should not break or crowd the UI.
    
    The iPhone and Watch sync path is much stronger now: faster HealthKit uploads, better Iroh/Tailscale routing, clearer sync status, recovery handling, and more reliable Watch command/action relay.
    
    Forge Memory also got safer update support and a Claude Code adapter, so Claude, Codex, Hermes, OpenClaw, and Forge Memory can share the same Forge runtime more consistently.
    
    Psyche and agent flows improved too. Agents should ask better questions, especially around beliefs, modes, triggers, behavior patterns, and meaningful records instead of just collecting fields.
