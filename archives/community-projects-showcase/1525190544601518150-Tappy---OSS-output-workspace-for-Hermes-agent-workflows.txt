    Just launched Tappy — an MIT-licensed open-source workspace/output layer for agents.
    
    The idea: agents like Hermes can start anywhere, but their work needs somewhere durable to land.
    
    Tappy gives agents a place to create and update real workspace objects instead of just leaving output in chat/logs/files:
    
    • docs
    • source-backed blocks
    • React/Python/code blocks
    • visible runs
    • reviewable changes
    • PDF/PPTX exports
    • BYOA workflows via MCP
    
    So a Hermes-style agent could do the work, then push the result into Tappy as editable/reviewable workspace state.
    
    The goal is not to replace the agent.
    
    It’s to give open agents a better output surface.
    
    Repo: https://github.com/MustafaK99/tappy
    Live: https://tappy.sh
    
    Would love feedback from Hermes/OSS agent people on what output primitives external agents should have next.
    
    allows for multi-agent collab so mulitple agents can collaborate together all self-hosted and free so you keep your own data
