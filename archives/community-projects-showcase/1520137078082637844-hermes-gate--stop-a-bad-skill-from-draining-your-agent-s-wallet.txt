    I run Observer Protocol and I've had my agent Atlas on Hermes for a few months, the persistent memory is what keeps me here.
    I kept Atlas research-only the whole time because I had no way to cap what an agent could spend if a skill went bad. So we built one: hermes-gate. Your agent calls it before any payment, it checks the spend against a mandate you signed (per-tx cap, rolling 24h cap, rail allowlist), and denies anything outside it. One command, installs via hermes mcp add, no core changes. Atlas runs it live, it's how I finally gave him a wallet yesterday!
    Honest about the boundary: today it's a check a compliant agent calls, so it stops a malicious skill pushing an honest agent past your limits but it doesn't YET stop an agent that bypasses the gate entirely. Making it unbypassable (the pre_tool_call hook looks like the right path) is what's next, and I'd love to compare notes with anyone who's worked at that layer.
    Repo: github.com/observer-protocol/hermes-gate
    
    Built it for this community — would value your eyes on it, especially where the threat model's wrong.
