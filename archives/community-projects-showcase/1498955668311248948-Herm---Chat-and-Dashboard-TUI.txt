    Before Hermes, OpenCode was my daily driver. I built Herm because I wanted Hermes capabilities with an OpenCode-like interface. Herm uses the same TUI framework OpenCode is built with, and also exposes dashboard style tabs that give easy access everything and anything I might want to do, all within my interface of choice: the terminal.
    
    I see a lot of Web UIs being made, but this is for the TUI lovers out there!
    
    https://github.com/liftaris/herm
    
    **11 Tabs**
    Chat · Context · Sessions · Agents · Analytics · Skills · Cron · Toolsets · Config · Env · Memory
    
    **Chat**
    - Streaming markdown, syntax-highlighted code, inline diffs
    - Animated ASCII avatar (.eikon format) with 6 agent states
    - Thought cloud overlay for reasoning tokens
    - Rewind / fork / branch from any message
    
    **Context Visualizer**
    - Every segment (system, skills, tools, memory, conversation) shown to scale
    - Drill into any segment to see what's in it
    
    **Look & feel**
    - 33 themes, all interchangeable with Opencode
    - drop any .json from opencode's theme registry into ~/.hermes/herm/themes/ and it works
    - Fully rebindable keys and shortcuts
    
    **Built on**
    - OpenTUI React renderer (same framework as OpenCode)
    - Bun runtime
    - Connects to Hermes the same  gateway the native hermes TUI does

    **1.4.0 release: fixes and QoL**
    TLDR:
    - 4 tab groups instead of 12 top-level tabs. Shift+←/→ for sub-tabs.
    - Consistent nav everywhere: Tab always walks fields/panes, Space toggles, hints live in a footer and reflect your rebinds.
    - Kanban diagnostics on cards: severity glyphs + suggested fixes in the side pane.
    - Profile distributions: i to install from a git URL, Info/Update from the profile menu.
    - Session reliability: /compress re-hydrates the transcript, /new/switch close the outgoing session cleanly, resume finds compression tips and cli-origin sessions.
    - Confirm on /clear /new /undo (opt-out via approvals.destructive_slash_confirm).
    - Snappier input: sidebar/chat/status no longer re-render on every keystroke.
