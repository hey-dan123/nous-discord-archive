    I built a small Hermes plugin for Human-in-Loop Camofox Authentication.
    
    It temporarily opens a native headed browser window or a private noVNC session, lets the user complete the login manually, and then resumes the task with the same persistent browser profile. The model only receives a confirmation that authentication was completed — not the credentials.
    
    ---
    **Why?**
    I ran into a common problem with GPT model: they can navigate pages, but they absolutely denies passwords, OTP/MFA codes, CAPTCHA answers, or other authentication secrets. And camofox cannot just open the browser in headed mode - either because it is on Remote Server or because Windows/macOS does not support useVirtualDisplay from Camofox. So, that's the solution
    ---
    
    The plugin provides a Human-in-the-Loop Authentication flow through Camofox. It supports local native mode on Windows/macOS/Linux and Docker + noVNC for VPS/server deployments.
    
    Install from GitHub through Hermes:
    
    ```shell
    hermes plugins install --enable tim-toothed/hermes-camofox-human-auth
    ```
