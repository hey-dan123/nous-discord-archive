    Built a self-audit skill for Hermes Agent to clean up the internal state drift that accumulates over time. Memory stores collect duplicate and contradictory entries, sessions that should have auto-closed stay open for weeks, cron jobs fail silently, and backup files pile up. None of it is catastrophic, but the agent gradually gets dumber as the cruft compounds. The skill audits memory, the session database, cron jobs, disk cruft, the session export script, and then verifies everything still works. It deduplicates memory, closes stale sessions, pauses failing jobs, prunes orphaned files, patches missing cleanup logic if needed, and generates a report in under a minute. The first run found 114 unclosed sessions going back two weeks, which turned out to be the root cause of Hermes re-answering messages and resurrecting shelved work. It now lives at ~/.hermes/skills/devops/self-audit/SKILL.md and is something I run whenever the agent starts acting haunted.
