    Built a memory provider plugin that integrates MyChatArchive (open-source, AGPL, local-first) as a Hermes backend.
    
    The wedge: every other provider builds the archive itself. This one assumes you already have years of cross-model chat history (ChatGPT, Claude, Cursor, Grok) and exposes it as Hermes memory. Read-heavy by design.
    
    Tested end-to-end via the Telegram gateway. 49 passing tests. About to open a PR on the main repo.
    
    Plugin lives at github.com/1ch1n/mychatarchive, integration code under integrations/hermes/.
    
    Two quick questions before I PR:
    1. Anyone aware of prior attempts at chat-archive-style memory providers I should reference?
    2. Is there a maintainer who handles memory provider PRs specifically?
