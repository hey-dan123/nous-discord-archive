    I’ve built a small Hermes plugin called Hora. (https://github.com/meleeislandbot/hermes-hora)
    
    Hora gives Hermes lightweight awareness of when user messages were received. It does this by adding an API-only temporal prefix to user messages right before the LLM call:
    
    ```[time: 2026-06-29T07:55:00+02:00] continue where we left off```
    
    The important part: this prefix is only sent to the model. It does not modify persisted transcripts, memory, or the visible chat history.
    
    The goal is simple: let the model naturally reason about time gaps in a conversation without adding complex temporal logic to Hermes core. For example, it can better infer when context may be stale, when a recap might be useful, or when “continue this” refers to something from minutes vs. days ago.
    
    Install:
    
    ```hermes plugins install meleeislandbot/hermes-hora --enable```

[2026-06-29T12:48:57.599000+00:00] henan (id=1521135387039367188)
    No. Hora only injects [time: ISO-8601] into the current user message at the llm_request middleware stage. It does not mutate the system prompt, prior messages, tool schemas, or persisted transcript, so it does not bust the reusable prompt-cache prefix. Cost impact is tiny: a few tokens per user turn.
