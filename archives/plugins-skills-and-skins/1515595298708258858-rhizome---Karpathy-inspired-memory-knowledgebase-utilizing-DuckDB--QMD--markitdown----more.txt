    Quick update on the repo with the latest pieces.
    
    Rhizome turns dropped documents into a living, searchable wiki that stays in plain markdown on your disk. Agents like Hermes, Claude, Codex, or OpenCode can all read and write to it. Knowledge builds up across tools without duplication.
    
    Recent updates:
    
    rhizome-research: multi-round deep research that pulls together sources and writes a clean wiki page for you
    rhizome-mcp: MCP server so any MCP client (Claude Code, Cursor, etc.) gets search, time-travel, lint, memory checks, graph, and eval tools with zero extra setup
    rhizome-graph: finds communities, suggests map-of-content pages, exports the graph
    rhizome-memory: tracks fact staleness and spots contradictions
    Full research with code context: point it at GitHub repos and it understands architecture before digging into the topic
    Local-first option with Ollama so even the writing step stays on your machine
    Core strengths:
    
    Compiles knowledge once into a persistent wiki instead of re-deriving answers from raw files every time
    Tracks how facts change over time (bi-temporal) so nothing gets lost or overwritten
    Ships an eval harness so you can measure how good the search actually is
    Everything stays as normal [[wikilinked]] markdown in your Obsidian vault
    Works with multiple agents at once and keeps everything local when you want it that way
