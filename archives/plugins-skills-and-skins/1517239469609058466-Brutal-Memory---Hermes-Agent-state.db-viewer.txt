    Brutal Memory — Hermes Agent state.db viewer
    
    A fully client-side web app for browsing all your Hermes Agent conversations. No backend, no uploads — just drag your state.db file into the browser and explore your session history.
    
    Features:
    - Drop & browse — drag a file, instantly see all sessions
    - Filter by source (tui, telegram, cron, discord, etc.)
    - Search by title with 300ms debounce
    - Sort by newest, oldest, most messages, or most tokens
    - Session details: model, source, duration, tokens, tool calls
    - Chat view with user/assistant/tool messages and JSON content parsing
    - Export single session as .md, all sessions as .json or .zip of .md files
    
    Stack: Vite 6 + React 19 + TypeScript + Tailwind v4.3 + sql.js (SQLite WASM) + zustand
    
    Design: Minimalist monospace — black background, white text, 2px borders, bracket-style buttons, zero border-radius.
    
    GitHub: https://github.com/brutal-build/brutal-memory
    Live demo: https://brutal-memory.vercel.app
    
    ```
    git clone https://github.com/brutal-build/brutal-memory.git
    cd brutal-memory
    npm install
    npm run dev
    ```
    
    Drop your state.db and see what your Hermes history looks like. Feedback welcome.
