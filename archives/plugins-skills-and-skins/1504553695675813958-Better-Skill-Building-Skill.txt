    The default skill-creator skill packaged with claude-code provides very low-quality guidance for skill creation, and leads to verbose skills with bad logic. Other skill-building skills also have similar problems, and are poorly thought-out. 
    
    To solve this issue, I created a logical framework as a skill, which replaces/complements these skills by providing logical framing for how to best structure skills for consistent quality and reliability.
    
    This skill defines skills in 3 tiers, allowing the agent to select the appropriate level of complexity for a skill it wants to build:
    
    Tier 1: `SKILL.md` (text-only)
    Tier 2: `SKILL.md + script(s)` (text + deterministic code)
    Tier 3: `SKILL.md + script(s) + AI/ML model` (text + deterministic code + purposefully trained specialized model)
