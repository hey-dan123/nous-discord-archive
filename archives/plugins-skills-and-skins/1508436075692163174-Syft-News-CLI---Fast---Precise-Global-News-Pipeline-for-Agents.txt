    I’ve been working on a way to **feed Hermes agents with high-fidelity, topic-specific news inputs** without relying on broad runtime web search.
    
    Instead of asking agents to scrape the web (slow/expensive/flaky), I built a CLI + Skill set that acts as an "intelligence layer." 
    It maintains a pre-recalled, topic-oriented pool and feeds Hermes structured artifacts via local JSON/Markdown.
    
    **What this adds to Hermes**:
    - Context injection: Inject fresh, curated topic-specific intelligence into your Agent’s session.
    - Skill-ready: Use the provided skills to transform noisy info into structured event timelines or source-labeled trees.
    CLI: https://www.npmjs.com/package/@orionarm/syft-cli (account creation and topic generation are needed on App first)
    GitHub: https://github.com/Solatrader/syft-news-skills
    
    Caveats:
    This isn't a native plugin yet (requires running the CLI to generate artifacts). I’m currently looking for feedback on whether this should be wrapped into a dedicated Hermes Plugin or an MCP server for better integration?
