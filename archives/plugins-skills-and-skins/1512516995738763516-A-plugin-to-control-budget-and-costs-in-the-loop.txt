    hermes-telemetry is an open-source observability plugin for Hermes Agent 
    that tracks token usage, estimated cost, latency, tool call count, and 
    session success/failure — all stored locally in SQLite with zero 
    infrastructure required.
    
    It hooks into Hermes' native plugin system (10 hooks registered) and 
    exposes your usage data through two slash commands: /stats and /budget. 
    Completely invisible to the model — no extra tools in the LLM's context.
    
    Built for: developers running Hermes agents locally or on a VPS who want 
    to understand and control their API spend without spinning up a full 
    observability stack.
    
    → GitHub: https://github.com/nujovich/hermes-telemetry

[2026-06-14T16:00:48.712000+00:00] nadiamermedev (id=1515747850162798622)
    **hermes-telemetry v0.4.1**
    
    Better cost accuracy across multi-provider setups, plus NVIDIA NIM works out of the box.
    
    **What's new**
    
    - **Provider-aware pricing guard** — pricing entries refreshed from one source no longer leak across providers. If two providers expose the same model id, each call is now billed at the rate of the provider that actually served it. Before, an OpenRouter-sourced entry could be applied to a Nous call and inflate recorded cost.
    
    - **NVIDIA NIM seed prices** — five NIM models ship with built-in pricing. NIM users get accurate `/stats` and budget enforcement on day one with zero config.
    
    - **Subscription flat-rate models are silent** — mark a model with `_subscription: true` in `pricing.yaml` and the plugin records $0 without the noisy "no price for model" warning. Good for Nous Portal flat-rate plans, Anthropic enterprise, NIM private endpoints, etc.
    
    **Backward compatibility:** none of your existing setups need changes. The new `provider` argument on `estimate_cost()` is optional.
    
    Run `hermes plugins update hermes-telemetry` to upgrade.
