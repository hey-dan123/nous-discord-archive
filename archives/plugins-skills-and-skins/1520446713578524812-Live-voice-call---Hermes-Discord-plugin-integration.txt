    ## ⚙️ Provider-flexible, not provider-trapped
    
    The current live voice path uses **Gemini Live**, but Hermes is the gateway layer.
    
    The goal is bigger than one model or one vendor:
    
    > **Voice → provider routing → tools → action → transcript → memory → follow-up**
    
    Current and emerging paths include:
    
    **Gemini Live** for full-duplex voice.  
    **opencode / codex / gemini / numasec / hermes-api** for CLI and agent delegation.  
    **GitHub / Home Assistant / Spotify / web tools** for real actions.  
    **Vapi sibling path** as an alternate voice/telephony direction, not the same thing as the Gemini Live bridge.
    
    ## 🔥 High-impact areas
    
    🔌 **Voice hackers** — Discord voice `4006`, reconnects, latency, `discord.py` internals.  
    🧩 **Tool builders** — Notion, Linear, Obsidian, Docker, email, calendar, file diff, repo actions.  
    🛠️ **CLI integrators** — opencode / codex / gemini routing, health checks, timeouts, fallback logic.  
    🔐 **Security reviewers** — `/frame` auth, sidecar hardening, transcript redaction, token/PII safety.  
    🐛 **Bug hunters** — long-call testing, weird edge cases, logs, reproducible issues.  
    📚 **Docs people** — make setup, diagrams, and contribution paths painfully clear.
    
    If you want to contribute to agent infrastructure while it is still being formed, this is the window.
