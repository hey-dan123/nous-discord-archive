    a Hermes plugin that lets external jobs, CI, automations, and agent runtimes send signed events back into the exact conversation where work started
    
    no cron polling, no hardcoded chat IDs, no manual nudges to your agent. Hermes owns the route, validates/de-dupes the event, boxes payloads as untrusted data, and either posts a notification or queues a bounded continuation back in-thread.
    
    it’s primarily tested today with Discord gateway threads, and solving the problem i had personally of trying to do long-running kanban orchestration from Discord, and wanting to be able to have my agent react to changes in the board, or tickets getting blocked, so it could orchestrate and create new tickets.
