    If you've hit the limits of Hermes' built-in memory (tiny char budget, no semantic search, resets between sessions) — Hexus is a plugin-based memory provider that fixes all of it.
    
    **What it is:** A drop-in memory provider plugin for Hermes Agent backed by PostgreSQL + pgvector. Stores embeddings locally via sentence-transformers — zero API cost, zero data leaving your machine.
    
    **Key features:**
    
    **Hybrid search** — blends vector similarity + full-text search + temporal decay. New memories weighted higher, old ones fade. Configurable weights per query.
    **Local embeddings (384-dim)** — runs MiniLM-L6-v2 via sentence-transformers on-device. No API keys, no external service, no data exfiltration. Falls back to Ollama or OpenAI-compatible endpoint if configured.
    **Entity graph** — tracks co-occurrence between entities. Ask "what docker images did we configure alongside Traefik?" and it walks the co-occurrence graph to find the answer.
    **CCR (Compressed Content Representation)** — auto-compresses large payloads before storing. JSON→key summaries, logs→error highlights, code→definition stubs. Keeps your context token budget sane.
    **Async writer** — non-blocking writes with a bounded queue. If the DB lags, it drops the oldest write instead of blocking the agent loop. Graceful degradation.
    **Conversation turn capture** — every substantive user/assistant exchange gets embedded. Full session recall across days and sessions.
    **Lifecycle webhooks** — HMAC-signed webhooks on startup, shutdown, and config changes. Hooks into monitoring and CI/CD.
    
    Try it out, give feedback, star the repo if you find it useful — anything is appreciated and welcome!
