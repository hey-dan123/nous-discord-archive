    Nika is not a Hermes skin and not exactly a skill
    
    It’s an open-source language + local runtime for turning repeatable AI/agent work into executable .nika.yaml files.
    
    The idea is:
    
    Hermes can reason, use skills and decide what should happen.
    
    Nika makes the procedure explicit:
    steps, model calls, tools/skills used, permissions, inputs/outputs, approval gates, and execution traces
    
    So instead of an important agent process living only in prompts, chat history, or custom glue code, it becomes a readable file you can version in Git, run locally, replay, and audit.
    
    Example use case:
    
    A Hermes business agent handles a customer issue:
    read ticket → check payment/order → decide refund or retry → ask approval if money moves → draft reply → save trace.
    
    With Nika, that whole flow can be written as one executable workflow file.
    
    Still pre-launch / early, but I’m looking for honest feedback so does this make sense as a companion layer for Hermes agents and skills?
