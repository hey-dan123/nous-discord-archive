    This was my experiment into reigning AI into writing disciplined, minimal code and eliminating AI bias by having other models review and scrutinize the decisions before any code is even written.
    
    🛂 HermesUltraCode — a pre-dispatch prompt gate for Hermes
    Built a Hermes plugin that vets every delegate_task before the worker subagent runs. The orchestrator's prompt doesn't reach the worker until a reviewer signs off — and that reviewer runs on a different model lab than your orchestrator.
    
    That cross-lab bit is the whole point: a model reviewing its own family's output shares its training data, its sycophancy, its blind spots — so it's structurally blind to the mistakes it's most likely to make. A genuinely different lab brings different failure modes and catches what a same-lineage reviewer would rubber-stamp. So: Hermes/Nous orchestrator → reviewed by Anthropic/xAI/whatever (you can even route the reviewer through hermes proxy). The gate hard-fails at startup if the labs match.
    
    hermes plugins install MahdiHedhli/HermesUltraCode
    
    (heads up: it fails closed, so set a reviewer key/lab before enabling — full flow in the README)
    
    Repo 👉 https://github.com/MahdiHedhli/HermesUltraCode

    How it wires in — real Hermes seams, no forking:
    
    ✂️ Tighten-only — the reviewer can only append constraints or block, never rewrite. dispatched = base verbatim + directives, enforced in code.
    🔒 Fail-closed — missing / timed-out / garbled verdict → block + escalate. Silence is never a pass.
    🧾 Immutable audit trail — one redacted row per dispatch, JSON/CSV export, plus a read-only dashboard (live state, gate panel, fail-closed counter).
    🧶 Neckbeard discipline — a vendored minimalism ruleset injected into the orchestrator + workers (forked from Ponytail, MIT). "Doesn't write less code, writes the correct code."
    It's a proper plugin — hooks tool_request (tighten) + pre_tool_call (block) on delegate_task, verified end-to-end against the real PluginManager. Stdlib-only core, 0 runtime deps, 155 offline tests.

    HermesUltraCode update — local-first, cost-aware model routing 🧠💸
    
    New drop for HermesUltraCode (the pre-dispatch gate that vets every delegate_task with a neutral, different-lab reviewer — tighten-only, fail-closed, audited). It now helps you run subagents on your own hardware and only pay cloud prices when a task actually needs it.
    
    🖥️ Use your local box. hermes ultracode-local probes your LM Studio / Ollama endpoint, lists the loaded models, and (with --apply) points subagents at it. Workers run local and free; your orchestrator + the cross-lab reviewer stay cloud, so the safety check is untouched. (Running Gemma-4-31B on a Mac Studio here — flies.)
    
    🧠 Cost-aware router (HERMESULTRACODE_ROUTING=1). Every released dispatch is scored: the cheapest model that actually clears the task’s required capability tier — local preferred via a tunable local_bias, but risk-gated off elevated / merge-authority work no matter how cheap, and capped so a quantized local model never takes the hard stuff.
    
    💸 See the savings. New Cost tab in the dashboard: dollars saved vs all-cloud, the local-vs-cloud split, and a per-dispatch table of what the router picked and why. (Demo run: 90% saved.)
    
    Honest caveat: routing is advisory today… it annotates and shows you the numbers, but Hermes binds the worker model from one global delegation: block, so true per-task binding needs a tiny upstream change (RFC in the repo, would love thoughts 👀). The config flip gives you the local win right now.
    
    hermes plugins install MahdiHedhli/HermesUltraCode
