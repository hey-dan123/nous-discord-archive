[2026-06-08T19:18:21.245000+00:00] capslock.b (id=1513623236095512656)
    <@113146055863840768> *what the gemini live bridge actually is, layer by layer:*
    
    platform adapter — discord voice client, sink/source pattern, 48khz stereo PCM in/out
    resampler — 48khz stereo ↔ 16khz mono with a FIR lowpass (this is what kills the "tin can" sound)
    model WSS — wss://generativelanguage.googleapis.com/... with a setup payload (model, generationConfig, voice, tools, systemInstruction) → server_content audio out
    tool execution + memory — function declarations, honcho peer per user, metrics, notes
    
    **the magic: layer 3 is transport-agnostic. the whole GeminiLiveBridge class is just "open WSS, send PCM in, get PCM out, dispatch function calls." you can swap layer 1 (discord) for an asterisk RTP listener and nothing else needs to change.
    
    **last thing — your setup is suspiciously similar to mine** (self-hosted dograh, private PBX, gemini live, pulling it into hermes).
