    /mcp + in-context catalog
    With a dozen MCP servers you either burn ~45k tokens of schemas every turn, or defer loading and the agent can't even see what exists. And mid-session your only lever is /reload-mcp or editing config + restart. This fixes both:
    
    🧩 In-context catalog (~0.5k tokens) — the agent always knows what servers exist (active / off / disabled) and proposes one when a task needs it; only you enable it. Plus a read-only mcp_catalog tool.
    🔀 /mcp list | on | off <name> — deterministic slash command (never goes through the model, works in the Discord picker). Toggles a server's tools live via agent.disabled_toolsets — persisted to config, effective next message, no restart.
    🔌 Control-plane only: never touches MCP transport, no second list to drift.
    
    Fills the open /mcp slash-command question from issue #690. Offline tests included.
    
    hermes plugins install pinkyhi/hermes-mcp-manager --enable
