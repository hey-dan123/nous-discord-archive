    game-development skill — PR: https://github.com/NousResearch/hermes-agent/pull/40796
    
    The gaming category had skills for playing games, but nothing for making them. 
    This fills that gap completely — an end-to-end game dev guide that takes you 
    from a blank page to a published game without needing any other tool.
    
    Built as an orchestrator + references, covering the full 11-stage lifecycle:
    concept → design → engine selection → architecture → core systems → gameplay 
    → art/audio → polish → optimization → QA → shipping → post-launch.
    
    Six deep reference files:
    - engine-selection — Unity vs Unreal vs Godot vs web, full decision matrix
    - core-systems — game loop, input mapping, state machines, event bus, save/load, pooling
    - unity — C#, MonoBehaviour, full player controller (coyote-time + jump buffer), AI FSM
    - unreal — C++/Blueprints, Gameplay Framework, reflection system, Enhanced Input
    - godot — GDScript, nodes & scenes, signals, CharacterBody2D controller, Resources
    - shipping — building for desktop/web/mobile + publishing to Steam, itch.io, mobile stores
    
    Engine-idiomatic, copy-and-adapt code for each engine — not pseudocode. Whether 
    you're making your first 2D platformer in Godot or shipping a 3D game to Steam, 
    it meets you at your stage.
    
    Triggers: "help me make a game", "implement a player controller in Unity", 
    "how do I publish my game to Steam"
