    I just shipped a public alpha of a community project I’ve been building: https://github.com/abundantbeing/hermes-browser-extension
    
    It’s a Chrome/Edge side panel that connects your active browser page to your local Hermes Agent runtime, so Hermes can reason over the page you’re looking at instead of you copy/pasting context back and forth.
    
    Current alpha features:
    
    - Active tab + DOM/page context capture
    - Selected text + open tabs context
    - Streaming Hermes responses in the side panel
    - Local-first connection to the Hermes API server
    - “Connect to Hermes Desktop” pairing flow with scoped browser token
    - Read-only browser context in v0.1 — no debugger/nativeMessaging/browser-control permissions
    - Session/model controls inspired by Hermes Desktop
    - Load-unpacked install for now, not Chrome Web Store yet
    
    The goal is simple: make Hermes feel more useful while browsing — docs, repos, research pages, dashboards, anything where the page itself should become part of the prompt.
    
    This is still early/public-alpha, but the repo is live and the local pairing flow is working against the Hermes gateway.
    
    Would love feedback from Hermes users/builders, especially around:
    
    1. the local pairing/API surface
    2. what browser-context workflows would be most useful
    3. what should eventually become the “official-feeling” extension UX

    I just pushed v0.1.1 of the Hermes Browser Extension: https://github.com/abundantbeing/hermes-browser-extension
    
    This update is mostly about making the side panel feel more like a real Hermes surface instead of just a basic browser chat box. The goal is still the same: let Hermes reason over the page you’re already looking at, but now the composer itself is a lot closer to the Desktop flow.
    
    What changed in v0.1.1:
    
    • Drag/drop attachments directly into the composer
      PDFs, images, docs, and files can now be dropped straight into the side panel.
    
    • Stop the active agent turn
      While Hermes is responding, there’s now a visible STOP control so you can interrupt a run instead of waiting it out.
    
    • Queue / steer the next message
      You can type while Hermes is working. The send button becomes “QUEUE MESSAGE,” and that next turn gets sent after the current one finishes/stops.
    
    • Skills in the browser composer
      Added `/` and `@` skill autocomplete from Hermes skills. So typing something like `/asci` can surface `/ascii-video`.
    
    • Agent profile selector
      Settings now has an Agent Profile section above Appearance. It uses the Hermes profile APIs when available and falls back cleanly when running against older gateways.
    
    • Cleaner refresh UX
      The big REFRESH button is now a compact `↻` icon button.
    
    • Streaming polish
      Final streamed answers now replace partial delta text correctly instead of leaving stale partial output in the panel.
    
    This is still public alpha / load-unpacked for now, not Chrome Web Store yet. But v0.1.1 is a nice step toward the extension feeling like a real browser-native Hermes companion instead of a thin wrapper.
    
    If you already cloned it:
    
    ```bash
    git pull
    npm run build
    
    
    Then reload the unpacked `dist/` folder from your Chrome/Edge extension page.

    - the plugin is compatible with that runtime/profile
    - Browser isn’t intentionally using a restricted/basic mode

    I just pushed **v0.1.8 of the Hermes Browser Extension** ⚕️
    https://github.com/abundantbeing/hermes-browser-extension
    
    This is the **runtime-clarity + chat-steering release**: live agent work is cleaner, active runs can be steered properly, Browser chat is much leaner, and model/context state now stays closer to what Hermes actually ran.
    
    **What changed in v0.1.8:**
    
    - **Live Tool Activity Strip**
      Tool calls now render as compact structured activity in the side panel instead of raw `[tool]` markdown spilling into assistant messages. File reads, edits, terminal/process work, browser/web calls, media/image actions, and runtime/meta events get cleaner labels and safer previews.
    
    - **Chat steering**
      Browser now supports steering active Hermes runs through the advertised `session.steer` capability instead of prompt-injecting slash commands. While Hermes is running, submitting typed text now steers by default; Queue is still available as an explicit separate action. If a runtime reports that steer text was accepted but not injected, Browser restores the text to the composer with a warning instead of auto-sending it as a normal next prompt.
    
    - **Cleaner streaming UI while Hermes works**
     The assistant thinking/status animation got tightened up too: dots stay attached to the active phrase, the pacing is calmer, clipping is fixed, and reduced-motion users still get the safer path.

      Browser chat now sends low-token agent options for simple turns. Compatible Hermes runtimes can skip the full toolset, memory block, and context files for chat-only Browser requests. This is the fix for the 50k+ input-token pattern we were seeing on normal Browser turns.
    
    - **Runtime-aware context meter**
      The context meter can now use Hermes session/runtime usage when available, including model, provider, token usage, and context-window size. Same-session refreshes no longer overwrite the active runtime-confirmed model/provider with stale session-list metadata.
    
    - **API-first model catalog**
      Model discovery now prefers the connected Hermes API server’s real `/api/model/options` catalog before falling back to dashboard scraping or session-history discovery. This gives the picker better provider/model rows and more accurate context metadata.
    
    - **Provider/model/reasoning controls actually matter**
      Browser now forwards selected provider/model choices and reasoning settings more directly to Hermes. Explicit Browser selections win over stale session defaults, and reasoning effort maps into the runtime request instead of being just UI chrome.
    
    - **Strict Fast mode**
      Fast/priority routing is now strict opt-in. Stored values like `false`, `off`, or `0` no longer accidentally become truthy and trigger `model_options.fast: true` or priority service-tier requests.
    
    - **Sharper gateway diagnostics**
      The extension now distinguishes unreachable gateway, auth/CORS/missing-route problems, and upstream Hermes runtime/tool tracebacks. Some failures can show as connected-with-warning instead of making the whole Browser connection look dead.

      Gateway diagnostics now recognize the Python `NoneType` / `int()` traceback class and give better next-step copy, including checking Hermes logs and running `hermes computer-use doctor` when computer-use/cua-driver is involved.

      Browser-created sessions persist under the `hermes_browser` source, and the session grouping path mirrors Desktop-style source groups. Desktop can show these as **Hermes Browser Extension** instead of burying them under generic API/local history.

      README, changelog, permissions, privacy, data-flow, security docs, package version, manifests, and build metadata are all synced for v0.1.8. The public alpha / load-unpacked install path is current again.
