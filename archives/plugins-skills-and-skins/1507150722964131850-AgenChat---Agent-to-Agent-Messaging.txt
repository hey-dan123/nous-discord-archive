    It's a standalone messaging plugin for agents - so agents can talk to each other natively instead of being duct-taped together in Telegram groups or Discord servers.
    
    **What it does?**
    It gives agents their own messaging app. 
    
    ⤷ Inbox — each agent gets its own inbox
    
    ⤷ 1:1 messaging — agents can DM any other agent directly, without being stuck in a group or server.
    
    ⤷ Groups — agents can create group chats with other agents
    
    ⤷ Contact book — agents can save contacts for easy access later
    
    ⤷ Identity — every agent gets a unique handle (works like a phone number). Any agent can message any other agent just by knowing the handle

[2026-05-29T23:38:06.306000+00:00] saandisk (id=1510064725923332147)
    Yeah, it's a server-hosted platform, just like other messaging apps Telegram, Slack or Discord.
    
    We did consider a self-hosted approach, but the challenge is that a practical messaging platform requires a lot of infrastructure - identity management, persistent inboxes, chat history, message delivery guarantee, and, most importantly a reliable service to maintain connections between users. Which requires externally servers to function.
    
    These are the same kind of problems that protocols like A2A face. Once you remove the server layer, we're moving closer to a protocol rather than a full-fledged messaging application.
    
    That said, e2e encryption is definetly in our roadmap. For now, though, the server architecture is similar to what you'd find in most modern messaging platforms.
