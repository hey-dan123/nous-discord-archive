    Tired of memory plugins that need a separate service or DB? Git as Memory stores everything directly in your repo's Git object store — no external dependencies, no cloud, no sqlite.
    Works with Hermes out of the box:
    npx skills add https://github.com/femto/skills --skill git-as-memory
    Each memory entry lives under refs/git-as-memory/memory/v1 — you can inspect, diff, and audit with standard Git tools:
    git log --oneline refs/git-as-memory/memory/v1
    git show refs/git-as-memory/memory/v1:memories/entity/user-preference-concise/memory.json
    delete writes a tombstone (history preserved), purge removes current files (Git history remains).
