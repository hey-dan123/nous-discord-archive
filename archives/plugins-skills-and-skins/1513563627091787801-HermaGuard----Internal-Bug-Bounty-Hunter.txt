    Most code review tools check for style violations and linting. Hermaguard tries to break your code.
    
    https://github.com/Sahil-SS9/hermaguard
    
        Three parallel subagents attack your diff from different angles: an edge case hunter that exhaustively traces every branching path in the change, an adversarial reviewer with a deliberately cynical persona ("your job is to break confidence, not validate"), and a blast radius mapper that traces every caller and callee to catch downstream breakage.
    
        Then a consolidator merges, de-duplicates, and triages everything into a risk-tiered report (CRITICAL/HIGH/MEDIUM/LOW).
    
        It's read-only — finds problems, doesn't touch your code. Opt-in only, no auto-trigger on commit. Run /hermaguard when you want to know what you missed.
    
        Built by synthesising 8 existing implementations: Trail of Bits differential review, BMAD's edge case hunter and adversarial reviewer, dementev-dev's Claude-Codex review loop, Anthropic's security review action, and the adversarial prompt pattern from r/ClaudeAI.
    
        PR is open: #42171
    
        If you write a skill that pairs with this (fix orchestrator, auto-test generator for findings, CI integration), open a PR. Bug reports in the review process appreciated. Complaints about the adversarial agent's attitude are also valid — it's designed to be a bit of a bastard.
