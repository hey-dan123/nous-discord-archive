    Built a small plugin after an agent spawned a batch of subagents on a job that didn't need them, and I wasn't expecting them; re-running the same job with delegation disabled produced almost as good a result for less than half the tokens:
    
    ### What It Does
    Intercepts every delegate_task call and routes it through the once/session/always/deny approval gate, showing the goal(s) and task count before you pay for subagents. An enviroment var `DELEGATION_GATE_MODE=block` refuses delegation outright and works with `--yolo` and on older runtimes that don't support the approve action yet (newer than 36308f066 needed for prompt mode).
    
    ### Why?
    There are config knobs to turn for delegation now but no way to request approval or simply deny it without removing the delegation toolset.
    
    ### Install
    Install for the default or a specific profile:
    - `hermes plugins install kenpritchard/hermes-delegation-gate --enable`
    - `hermes -p myprofile plugins install kenpritchard/hermes-delegation-gate --enable`
    
    ### Usage
    Standard usage simply let the agent work and prompt you for approval to delegate tasks. Try a prompt like this:
    ```
    Use delegate_task to list the contents of this directory and return the count. Do not do it yourself.
    ```
    
    If running YOLO and you want it to block, start your agent with the env var set:
    ```
    DELEGATION_GATE_MODE=block hermes -p myprofile --yolo
    ```
    
    Other ways to set the env var including Windows are in the readme in the repo.
