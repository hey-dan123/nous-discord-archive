    Just shipped a fix for the WSL2 OAuth callback issue a lot of folks have been hitting on hermes auth add xai-oauth. 
    
    The "Could not establish connection" page is from xAI's authorize server when the Windows browser → WSL loopback redirect gets dropped by the default Hyper-V Firewall block on the WSL VM. 
    
    Repo has a no-admin wrapper script (scripts/hermes-xai-oauth-wsl.sh) and a one-time admin PowerShell rule (docs/permanent-fix.ps1). 
    
    MIT, PRs welcome — especially anyone who wants to land a --code flag upstream so this wrapper isn't needed. https://github.com/jettoptx/hermes-xai-oauth-wsl
