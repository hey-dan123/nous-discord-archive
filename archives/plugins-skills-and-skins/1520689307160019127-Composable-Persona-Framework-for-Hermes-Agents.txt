    I’ve added Chinese and Korean speech styles.
    Additionally, I’ve added a new layer called “use_case / Operating Mode” following “personality / speech.”
    This allows you to add professional-level rules tailored to specific use cases while maintaining the same personality.
    
    Example:
    $ hersona blend tsundere keigo --use-case programmer
    
    The initial four types:
    - programmer
    - planner
    - research
    - marketing
    
    The control plane is fixed in English (for token efficiency and better compliance with LLM control commands).
    It is fully compatible with existing behavior and can be upgraded using `pip install -U hersona`.

    This wont destroy the prompt caching, right?

[2026-07-04T18:54:27.257000+00:00] shiro1001 (id=1523039306019639386)
    No—Hersona itself does not interfere with prompt caching.
    Hersona is a prompt composition library. The recommended workflow is to compose the persona once (or only when it actually changes), and then reuse that generated system prompt throughout the conversation.
    If you rebuild the full system prompt on every single request, the cache efficiency will naturally drop. However, this is true for any prompt generation method, not a problem specific to Hersona.
    Thanks for asking! It's a great question.
