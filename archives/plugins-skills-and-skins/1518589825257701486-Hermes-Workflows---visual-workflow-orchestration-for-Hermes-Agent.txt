    Describe a process as a graph and run it on Hermes' own primitives. It is a thin layer, not a new engine - workflows compile straight to native Hermes Kanban tasks, Cron jobs, and Profile assignments. It does not replace any of them.
    
    • Headless-first: a run advances on a self-terminating Cron tick, no human in the loop except an explicit human_review node
    • Node types: trigger, agent_task, script, condition, human_review, finish
    • Scope picks the backend: a project run schedules durable Kanban cards, a global run invokes the profile runner directly
    • Visual editor on @xyflow/react: author the graph, validate, preview the compiled Hermes plan, then run and inspect per node, with Runs and Schedules views
    • Optional Open Second Brain for long-term memory
