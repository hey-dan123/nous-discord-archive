    I built a drop-in MemoryProvider for hermes-agent that gives an agent a **real** long-term memory instead of a flat log that grows forever. The core idea is that facts behave like resonant circuits. A memory that keeps getting struck stays in tune and rings loud, so it gets reinforced and promoted into a durable tier. The ones that nothing resonates with fade to silence and eventually get pruned. The whole system  is driven by usage cycles (think dream cycles that decay, promote, abstract, and resolve conflicts), not by wall-clock timers, so it keeps what matters and forgets the noise on its own.
    
    The thesis behind it: for an agent that is expected to earn, spend, or runs real operations, memory is the load-bearing wall. It has to do four things well. Recall the right thing across sessions, never fabricate a fact, obey the standing rules you pin into it, and resist being poisoned or going stale. The system is built and tested around exactly those four properties.
    
    Under the hood it is a three-tier resonance store (short, mid, long) over SQLite plus sqlite-vec, with holographic (HRR) compositional encoding and an entity graph. Recall is hybrid (vector plus keyword) with a precision gate and an authority preference for pinned facts. Anti-fabrication is enforced by source-quote attestation, and the agent can influence memory (reinforce, pin, give feedback) but cannot silently destroy it. It runs local-first on Ollama: a small embedding model plus any reasoning model you like for the off-hot-path consolidation. There is also an experimental encryption tier (at-rest plus a homomorphic blind-recall preview) if you want it.

    There is also a first-class (but still maturing) homomorphic blind store mode you can enable via `encryption_mode: blind`. This was built to realize the deeper goal of functional sovereign memory where:
    
    - Only the primary owner ever holds the master secret (derived from a passphrase via Argon2id + HKDF).
    - The untrusted store node only ever sees public and evaluation keys.
    - A delegated agent only ever receives *re-encrypted* results via proxy re-encryption (PRE) that it can decrypt with its limited use-key — it cannot read the raw store.
    
    Both Tier 0 (SQLCipher encrypted-at-rest) and Tier 1 (CKKS homomorphic blind store) are implemented and working. The blind tier already supports:
    
    - Encrypted embeddings (`semantic_he`) and HRR lifts (`semantic_he_hrr`) using separate CKKS contexts.
    - Scaled resonance maintenance (`semantic_he_meta`).
    - AEAD-encrypted entity name sets with client-side overlap detection (`semantic_he_entities`).
    - Proxy re-encryption of recall scores so the store can return results without ever decrypting them.
    - Per-cycle scope bounding (`ScopeLimiter` + re-encryption audit log) to prevent a compromised agent from exfiltrating the entire memory.
    
    I have this working well on the node (including write-path completeness and the multi-keyset keystore), but the remaining experimental pieces are:
    
    - Autonomous blind dream-cycle maintenance using the lightweight (depth-1) maintenance context + decay-from-origin (the current path still leans on the deeper context for in-place decay).
    - Full integration of pure-CKKS blind argmax / top-k into the live recall path (so the store can return only winning facts without the client learning the full ranking).
    - Production hardening of the comparison circuits (security level + bootstrapping) and tighter wiring of the scope/re-encryption policy enforcement into the hot path.
    - Key rotation, sealed storage (TPM / once-per-boot unlock), and multi-user delegation stories.
