    I built a super light weight Hermes Agent plugin that gives your agent a persistent character identity. It survives sessions, restarts, and context resets.
    
    It's a five-layer personality stack:
    
    SOUL.md
     — core identity (who your character is)
    /personality
     — how they think, feel, and speak
    Prefill
     — few-shot behavioral anchor (survives context drift)
    persona-config.json
     — dynamic rules, time-slots, relationship dimensions
    Memory
     — what they learn across sessions (MCP memory providers)
    
    Changes hot-reload on save. No restart needed. 🔁
    
    The repo ships with example configs so you can see how it works immediately — including a black cat named Luna who ends every sentence with "nya~" 🐱 Running in 30 seconds from install.
    
    hermes plugins install kenyonxu/hermes-persona --enable
    
    
    Repo: https://github.com/kenyonxu/hermes-persona
    Docs in CN + EN. Pick a character or build your own — the five layers fit anything.

[2026-05-26T23:47:46.627000+00:00] kai.xu (id=1508979996369227806)
    Hey! You've hit on something real — it's less a chat-template issue
    and more about training-side specialization.
    
    Kimi and DeepSeek likely saw heavy roleplay fine-tuning, so they
    treat system-level persona instructions as a strong signal. Qwen
    leans instruction-following, Llama is more general-purpose, and
    Gemma's safety guardrails can sometimes suppress strong identity
    directives — the model isn't "ignoring" soul.md, it's just not wired
    to prioritize it the same way.
    
    Two practical options:
    
    1. Without a plugin: multi-point anchoring. Don't rely on system
       prompt alone — inject key persona rules into the first user
       message as well, and reinforce in prefill. When the same "who
       you are" signal appears in multiple positions, Qwen/Llama/Gemma
       treat it more like a hard constraint.
    
    2. With hermes-persona (the plugin from the repo you're commenting
       on): its persona-config lets you distribute soul-level rules
       across different injection points in context — system, prefill,
       per-turn append. That way you're not betting everything on
       system-role obedience. Works especially well for models that
       need the persona "ribbon-woven" rather than declared once.
    
    hope this helps！ps: I personally use kimi and deepseek, too
