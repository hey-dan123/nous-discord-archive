    🕒 **A real timeline of your agent's beliefs**

[2026-05-29T07:20:05.870000+00:00] techmeat (id=1509818602336026664)
    Hey! The plugin has grown quite a bit since this post — several updates have shipped (from 0.15.0 through 0.19.0). Here's a quick rundown of what's new and why it matters:
    
    **🧩 You can now see who "taught" the brain what**
    Ask what Claude, Codex, or Hermes each contributed and compare them side by side — spot where one agent has context another doesn't. Memory stopped being a shared "dump" and became a map: who suggested which rule and where topics overlap.
    
    **🛡️ Personal space and cleaner boundaries arrived**
    You can pin temporary notes for the current session (they won't turn into permanent memory), and wrap sensitive text in private blocks that get stripped out before anything is saved long-term. Links can now adapt to the client too: plain Markdown where Obsidian-style links aren't needed.
    
    **🩺 Tidying up and "reviewing" memory got easier**
    New review surfaces help you sort through what's ready to process, what's gone stale, and what to keep, improve, or prune — all as recommendations, with no auto-deletion. There's a monthly overview of events, plus a smart warning when your vault's structure grows faster than the actual value it brings.
    
    **💾 Big responses no longer flood your context**
    Heavy results now come back as a short preview, with the full version available on demand — nothing is lost. Search also gained a one-line hint so you can tell what was found at a glance.
    
    **🔍 Links between notes got smarter**
    You can now mark the type of a connection — whether something supersedes, extends, or contradicts another note — and search takes that into account. You can also hide certain notes from regular search and only surface them when needed.
    
    Overall: more transparency, more control over privacy, and a noticeably smoother experience when working with large amounts of memory. 👍

    Hey everyone 👋 Another batch of updates landed since the last post (v0.19.0 → v0.32.x). The big one this time is native Hermes support 👇
    
    🧠 **Native Hermes memory provider - the headliner**
    Open Second Brain now plugs into Hermes as a first-class memory provider, not a bolt-on. A single registration opens the full Hermes memory lifecycle: `Brain/active.md` lands in the system prompt, every turn passes a recall gate before the model call, completed turns feed the nightly dream pass, and Hermes' own `MEMORY.md`/`USER.md` writes mirror straight into your Brain. Activate with `hermes memory setup open-second-brain` - and it persists across plugin updates, no re-activation needed. Claude Code and Codex keep their MCP path unchanged.
    
    🔌 **Full compatibility - upstream PR**
    To make external providers with relative imports load fully natively (the same path bundled providers use), I opened a small fix upstream: NousResearch/hermes-agent#37366 - <@387972437901312000> would love your eyes on it 🙏 The plugin already works on a stock Hermes install today via a self-bootstrap loader; the PR just removes the last workaround.
    
    A few other things along the way: self-learning **procedural memory** (the brain now learns *how* work repeatedly gets done, review-first), a **context-safety layer** that screens untrusted note content before it reaches an agent, **environment-based credential pointers** so sensitive values stay outside the vault, **recall control / evidence packs** so you can see *why* something was surfaced, **CJK search**, and **hands-off post-upgrade migrations** (updates never need manual steps).

[2026-06-05T11:23:15.270000+00:00] techmeat (id=1512416509774008331)
    Hey all 👋 Another batch landed since the last post (v0.32.x → **v0.45.0**). The big one this time: the upstream fix is **in Hermes** 🎉
    
    🧠 **Native Hermes provider - the workaround is gone**
    Until now the plugin loaded on Hermes via a self-bootstrap shim - a workaround, because Hermes' loader didn't register the parent namespace for user-installed providers, so anything with relative imports (like our subpackage layout) wouldn't load the documented way. I opened a small upstream fix for exactly that https://github.com/NousResearch/hermes-agent/pull/37366, and it's now cherry-picked into main via #38995 (thanks <@387972437901312000> ). So **Open Second Brain is now a 100% first-class memory provider exactly as the docs describe** - no shim, `hermes open-second-brain ...` discoverable on a stock Hermes out of the box, and it persists across plugin updates.
    
    Huge help during development was the memory-provider plugin guide: https://hermes-agent.nousresearch.com/docs/developer-guide/memory-provider-plugin
    
    **What's new since v0.32.x:**
    
    🕒 **Time-aware recall** - activation decays by content type + event-time windows, so old facts stop polluting answers
    
    📒 **Entity truth ledger** - current-truth per entity with conflict detection, plus a **dead-end registry** for negative knowledge
    
    🌳 **Git-history → memory** - ingest commits/releases, answer "why did this change", commit→ADR miner
    
    🔐 **Secret custody** - AES-256-GCM at rest, stdin-only ingestion, allowlisted exec, no values in logs
    
    ✍️ **Write-time integrity** - schema-pack ontology: controlled-vocab labels, link constraints, frontmatter tiers
    
    🔗 **Link & recall intelligence** (newest) - alias-resolved wikilinks as real graph edges, reviewable bridge discovery, graph communities
    
    📊 **CI-gated recall benchmark** - hit@k + MRR fails CI on ranking regressions, plus opt-in self-tuning recall
