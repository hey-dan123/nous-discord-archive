[2026-05-30T11:36:01.149000+00:00] vayaperson (id=1510245394909499443)
    We're almost there!
    
    We're making the final adjustments to the new features to ensure they work correctly and are as reliable and secure as possible (while always keeping in mind that bugs can still exist. Everyone should be aware of how they use the skill and on which systems, with access to what information, they choose to run it).
    
    We're currently working on the following issues:
    
    1. **Bot LID resolution error**
       The bot's LID is not being resolved correctly, causing user messages and bot responses to appear as two separate conversations in the inbox.
    
    2. **Session fragmentation = amnesia effect**
       Because the LID is not being resolved properly (see point 1), JID extraction fails, resulting in an empty `session_key`. Memory itself is working, but session fragmentation combined with the LID resolution issue causes this behavior.
    
    3. **Souls behaving like generic assistants**
       Personalities are being applied correctly, but when a subsoul is loaded it is currently added as additional context. This means the model may remain anchored to its original role if the override is not reinforced in a more effective way.
    
    4. **Permissions and Knowledge system refactor**
       We're restructuring the permissions system. Currently, if a user only has permission to interact with the bot, any request requiring access to the Knowledge system may be rejected because the user lacks file read permissions. We're also redesigning how command flags/arguments are handled and how their usage is managed through permissions.
    
    5. **Spaces in soul names**
       A simple parsing issue with an easy fix.
    
    We'll have everything sorted out soon!! Stay tuned!! 👀

    📱 Admin recognition over WhatsApp
    Sometimes the system wouldn't recognize the owner when sending commands via WhatsApp. This was just a quirk with how WhatsApp handles device IDs (LIDs) under the hood.

    We noticed that in very long conversations, the bot could sometimes start drifting out of its assigned persona.
    
    The fix:
    Translated the core identity prompt to English (which works much better with most models).
    Tweaked the prompt order so the operational mode has higher priority.
    Added a subtle "soul reminder" that refreshes the persona every 8 turns. This keeps the bot perfectly in character even in endless chats.

    **Update on what we actually found: it was a bit more than we thought**
    
    So after digging deeper today, turns out the issues in Beta 3 were a bit more layered than our initial post suggested. Wanted to be transparent about what we actually found vs. what we said.
    
    Admin recognition - the real story: We said it was a LID/contacts cache issue. Partially true — but the actual root cause was simpler and sneakier: the ADMIN_PHONE in the config doesn't include a country prefix (xxxxxxxxx), but WhatsApp sends the full number (34xxxxxxxxx). The comparison was failing silently. We also found a second issue: The JID section in the rules correctly marked someone as owner, the system wasn't checking that as a fallback. It relied on the phone number match. Both are now fixed. Additionally, the admin panel UI had a race condition where setting "Hermes Native" soul would visually snap back to "Default Soul" on reload (patched too).
    
    Long-term memory was offline 🧠 This one we didn't mention at all: Hindsight (memory layer backed by PostgreSQL) was silently not running. The process had died and left a stale PID. The bot was working fine short-term but had zero cross-session recall. We've fixed the startup, added a compatibility layer for the library version mismatch, and added a visible warning banner in the admin panel so this never goes unnoticed again — with a one-click "Start Memory" button.
    
    Soul/persona stability: No corrections needed here.
    
    Soul transition - One thing we didn't mention before: when you swap a contact's assigned soul, the system was wiping the entire conversation history. Now instead of a full wipe, it preserves the user's messages for context continuity but strips the bot's previous responses so new soul picks up the conversation context without being anchored to the old persona's voice.
    
    Sorry for the vague initial post. We were mid-investigation. We are testing the fixes. Beta 4 drop soon.
